# スマホなし子ども画面開発 - クイックリファレンス

**対象:** スマホなし子ども画面の担当開発者
**用途:** デスクに貼っておく用の簡易チェックシート

---

## 🚨 3大禁止事項

### 1. mainブランチへの直接push 🚫

```bash
❌ git checkout main && git push origin main
✅ git checkout -b feature/kids-screen && git push origin feature/kids-screen
```

### 2. 共通ロジック（lib/hooks/）の変更 🚫

```typescript
❌ 既存関数を変更: export const fetchStampCount = async ...
✅ 新しい関数を追加: export const fetchChildStampCount = async ...
```

### 3. データベーススキーマの変更 🚫

```sql
❌ CREATE TABLE / ALTER TABLE
✅ メイン担当者に相談
```

## ❌ 編集禁止フォルダ

```
components/(adult)/            ❌ 大人用画面
app/api/                      ❌ 既存API（相談なしで変更禁止）
lib/                          ❌ 共通ロジック（相談なしで変更禁止）
hooks/                        ❌ 共通hooks（相談なしで変更禁止）
supabase/                     ❌ データベース
contexts/                     ❌ 共通Context
types/                        ❌ 型定義（相談なしで変更禁止）
```

---

## 🔄 Git運用フロー

```bash
# 1. ブランチ作成
git checkout -b feature/kids-screen

# 2. 開発・テスト
npm run dev

# 3. コミット（必ず (child-mode) を付ける）
git add app/child-mode/
git commit -m "feat(child-mode): ○○機能を追加"

# 4. Push
git push origin feature/kids-screen

# 5. GitHub上でPull Request作成
# → ソラシドワークス様がレビュー
```

---

## ✅ Push前チェックリスト

- [ ] `npm run build` が成功する
- [ ] `npx tsc --noEmit` でエラーなし
- [ ] `app/child-mode/` または `components/(kids)/` のみを変更
- [ ] コミットメッセージに `(child-mode)` を付けた
- [ ] 不要なファイル（`.next/`, `node_modules/`）をコミットしていない

---

## 🆘 困ったときは

以下の場合は**作業を止めて**メイン担当者に相談：

1. ✋ `KidsHome.tsx` を大幅に変更したい
2. ✋ `lib/` や `hooks/` の既存関数を変更したい
3. ✋ 新しいAPIエンドポイントが必要
4. ✋ データベースに新しいテーブル/カラムが必要
5. ✋ ビルドエラーが解決できない
6. ✋ 新しいライブラリをインストールしたい
