# イベントログ記録問題のデバッグ手順

## 📋 問題の状況

### 記録されているログ ✅
- `app_open` (AppLayout.tsx から呼び出し)
- `stamp_page_view` (AdultStampPage.tsx から呼び出し)

### 記録されていないログ ❌
- `stamp_scan_start` (app/scan/page.tsx から呼び出し)
- `stamp_scan_api_request` (app/scan/page.tsx から呼び出し)
- `stamp_scan_success` (app/api/stamps/scan/route.ts から呼び出し)

## 🔍 調査ポイント

### 1. RLS（Row Level Security）は機能している
**根拠**: `app_open` と `stamp_page_view` が記録されている
- RLSポリシー: `with check (true)` で全ユーザーが挿入可能
- 結論: RLSの問題ではない

### 2. クライアントサイド vs サーバーサイド

| ログ種別 | 呼び出し元 | 環境 | 状態 |
|---------|----------|-----|------|
| `app_open` | AppLayout.tsx | クライアント | ✅ 記録される |
| `stamp_page_view` | AdultStampPage.tsx | クライアント | ✅ 記録される |
| `stamp_scan_start` | app/scan/page.tsx | クライアント | ❌ 記録されない |
| `stamp_scan_api_request` | app/scan/page.tsx | クライアント | ❌ 記録されない |
| `stamp_scan_success` | app/api/stamps/scan/route.ts | サーバー | ❌ 記録されない |

### 3. 考えられる原因

#### 原因A: クライアントサイドのエラー（スマホでは確認不可）
- ブラウザコンソールにエラーが出ているが、スマホLIFFアプリでは確認できない
- `logEvent()` は try-catch で囲まれており、エラーを console.error に出力するが、ユーザーには見えない

```typescript
// lib/analytics.ts:109-112
catch (error) {
  console.error('⚠️ Analytics error:', error);  // ← スマホでは見えない
  return { success: false, error: String(error) };
}
```

#### 原因B: サーバーサイドのエラー（Vercelログで確認可能）
- API Route (`app/api/stamps/scan/route.ts`) でのエラー
- Vercel Function Logs で確認可能

#### 原因C: 関数が実際に呼ばれていない
- 条件分岐でスキップされている
- await が抜けている（fire-and-forget になっている）

## 🛠️ デバッグ手順

### Step 1: Vercelログを確認（サーバーサイド）

1. **Vercel ダッシュボードにアクセス**
   - https://vercel.com/
   - プロジェクトを選択
   - 「Logs」タブをクリック

2. **QRスキャン実行時のログを確認**
   - QRスキャンを実行
   - Vercelログで `/api/stamps/scan` のリクエストを探す
   - 以下のログが出ているか確認:
     ```
     ✅ QRスキャンスタンプ登録成功: User xxx, Type: premium, Stamps: +15個
     ```
   - エラーログが出ていないか確認:
     ```
     ❌ Analytics error: ...
     ```

## 🎯 解決策の候補

### 解決策1: エラーハンドリングを強化

```typescript
// lib/analytics.ts を修正
export const logEvent = async (params: LogEventParams): Promise<LogEventResponse> => {
  try {
    // ... existing code ...

    const { error } = await supabase.from('event_logs').insert({
      user_id: targetUserId || null,
      event_name: eventName,
      source: detectedSource,
      metadata: enrichedMetadata,
    });

    if (error) {
      // サーバーサイドではVercelログに記録される
      console.error('❌ [Analytics] Event log insertion failed:', {
        eventName,
        userId: targetUserId,
        error: error.message,
        details: error,
      });
      return { success: false, error: error.message };
    }

    // 成功時も明示的にログ
    console.log('✅ [Analytics] Event log inserted:', {
      eventName,
      userId: targetUserId?.substring(0, 8) + '...',
    });

    return { success: true };
  } catch (error) {
    console.error('⚠️ [Analytics] Unexpected error:', {
      eventName: params.eventName,
      error: error instanceof Error ? error.message : String(error),
      stack: error instanceof Error ? error.stack : undefined,
    });
    return { success: false, error: String(error) };
  }
};
```

### 解決策2: API Routeでエラーを確実に記録

```typescript
// app/api/stamps/scan/route.ts
try {
  await logStampScanSuccess({
    stampsAdded: stamps,
    type: type,
    userId: userId,
    currentStampCount: currentStampCount,
    newStampCount: finalStampCount,
    stampHistoryId: stampData?.id,
    milestonesGranted: grantedRewards,
  });
} catch (logError) {
  // イベントログ失敗してもスタンプ付与は成功しているので処理続行
  console.error('❌ [API] イベントログ記録失敗（スタンプ付与は成功）:', logError);
}
```

### 解決策3: クライアントサイドのログをサーバー経由で送信

クライアントサイドのログをAPI経由で送信することで、エラーをVercelログで確認可能に:

```typescript
// app/api/analytics/log/route.ts （新規作成）
export async function POST(request: NextRequest) {
  const body = await request.json();
  const result = await logEvent(body);
  return NextResponse.json(result);
}

// app/scan/page.tsx から呼び出し
await fetch('/api/analytics/log', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    eventName: 'stamp_scan_start',
    userId: profile?.userId,
    metadata: { device_type: deviceType },
  }),
});
```
