## 概要

### 🎯 目的

**「標準カメラでQRコードを撮るだけで、LIFFアプリが起動し、自動的にスタンプが付与される」**

現在の「アプリ内でカメラを起動する」という二度手間をなくし、受付でのスタンプ付与を**3秒以内**に完結させる。

### ✨ 期待される効果

| 項目 | 改善効果 |
|------|----------|
| **操作時間** | 15〜20秒 → **3秒** |
| **受付滞留** | ほぼゼロに |
| **スタッフ説明** | 不要（「ポスターを撮ってください」のみ） |
| **子供の体験** | 「楽しいイベント」に変化 |
| **不正防止** | 1日1回制限で対応可能 |

## 現状の課題

### ❌ 現在のフロー（5ステップ、約15〜20秒）

```
1. LINEアプリを探して開く（5秒）
   ↓
2. ミニアプリを探してタップ（3秒）
   ↓
3. アプリが起動するのを待つ（3秒）
   ↓
4. 「スキャン」ボタンを探してタップ（2秒）
   ↓
5. カメラが起動し、QRコードをスキャン（3秒）
   ↓
6. スタンプ付与完了
```

**合計**: 約15〜20秒 + スタッフの説明時間

### 🚨 ユーザーの心理的ハードル

1. **「LINEどこ？」問題**
   - アプリを探すのが面倒
   - LINEアイコンが見つからない

2. **「ミニアプリって何？」問題**
   - ミニアプリの概念が分からない
   - どこにあるか分からない

3. **「操作が多い」問題**
   - ステップが多すぎて覚えられない
   - 次回来院時に忘れている

4. **「受付で混雑」問題**
   - 後ろに人が並んでいるプレッシャー
   - スタッフを呼び止めるのが申し訳ない

## 撮るだけスタンプの仕組み

### ✅ 改善後のフロー（1ステップ、約3秒）

```
1. iPhoneの標準カメラでポスターのQRコードを撮る
   ↓
   （自動的にLINEブラウザが起動）
   ↓
   （自動的にLIFFアプリが起動）
   ↓
   （URLパラメータを検出）
   ↓
   （自動的にスタンプ付与API実行）
   ↓
2. 「スタンプを押しました！」完了画面が表示される
```

**合計**: 約3秒 + スタッフの説明「ポスターを撮ってください」

### 🎯 技術的ポイント

**URLパラメータによる自動判定**

```
通常のURL:
https://liff.line.me/2009075851-74EieWb4

アクション付きURL:
https://liff.line.me/2009075851-74EieWb4?action=stamp&type=qr&amount=10
```

アプリ起動時（`useEffect`）に、URLパラメータをチェック：
- `action=stamp` → スタンプ付与処理を自動実行
- `type=qr` → QRスキャンによる付与と記録
- `amount=10` → 10個付与（通常来院）

## URL構造の設計

### 基本構造

```
https://liff.line.me/2009075851-74EieWb4?action=stamp&type=qr&amount=10&location=entrance
```

### パラメータ一覧

| パラメータ | 必須 | 値 | 説明 |
|-----------|------|-----|------|
| `action` | ✅ | `stamp` | スタンプ付与アクション |
| `type` | ✅ | `qr` / `manual` / `slot` | 付与方法 |
| `amount` | ✅ | `5` / `10` / `20` | 付与個数 |
| `location` | ⭕ | `entrance` / `reception` / `treatment` | QRコード設置場所 |
| `campaign` | ⭕ | `welcome` / `birthday` / `event` | キャンペーンID |
| `token` | ⭕ | ランダム文字列 | セキュリティトークン（将来） |

### URL例

#### 1. 通常来院（10個付与）
```
https://liff.line.me/2009075851-74EieWb4?action=stamp&type=qr&amount=10&location=entrance
```

#### 2. キャンペーン（5個付与）
```
https://liff.line.me/2009075851-74EieWb4?action=stamp&type=qr&amount=5&campaign=welcome
```

```typescript
const handleAutoStamp = async (params: {
  type: string;
  amount: number;
  location?: string;
  campaign?: string;
}) => {
  try {
    // 1. LIFFの初期化を待つ
    await liff.init({ liffId: '2009075851-74EieWb4' });

    // 2. ユーザー情報を取得
    const profile = await liff.getProfile();

    // 3. 1日1回制限チェック
    const canAdd = await checkDailyLimit(profile.userId);
    if (!canAdd) {
      showMessage('今日はすでにスタンプを付与しました');
      return;
    }

    // 4. スタンプ付与API実行
    const result = await fetch('/api/stamps/auto', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        userId: profile.userId,
        type: params.type,
        amount: params.amount,
        location: params.location,
        campaign: params.campaign,
      }),
    });

    // 5. 完了画面を表示
    if (result.ok) {
      showSuccessScreen(params.amount);
    } else {
      showErrorScreen();
    }
  } catch (error) {
    console.error('自動スタンプ付与エラー:', error);
    showErrorScreen();
  }
};
```

## セキュリティと不正防止

### 🚨 想定される不正パターン

#### 1. 家でQRコードの写真を撮って毎日付与
**対策**: 1日1回制限

```typescript
// 同一ユーザーが同一日に複数回付与しようとした場合
const { data: todayStamps } = await supabase
  .from('stamp_history')
  .select('*')
  .eq('user_id', userId)
  .eq('stamp_method', 'qr_scan')
  .gte('visit_date', `${today}T00:00:00Z`)
  .lte('visit_date', `${today}T23:59:59Z`);

if (todayStamps && todayStamps.length > 0) {
  return { success: false, message: '今日はすでにスタンプを付与しました' };
}
```

#### 2. URLを直接ブラウザで開いて付与
**対策**: LIFF SDK の認証チェック

```typescript
// LIFFの初期化が完了していない場合はエラー
if (!liff.isLoggedIn()) {
  throw new Error('LINEログインが必要です');
}

// LINEユーザー情報が取得できない場合はエラー
const profile = await liff.getProfile();
if (!profile || !profile.userId) {
  throw new Error('ユーザー情報が取得できませんでした');
}
```

#### 3. URLをコピーして他の人に配布
**対策**: 特に対策不要（1人1日1回制限で十分）

実害: 限定的（最大でも1日1回のみ）

理由: URLは誰でも見られるが、結局LINEアカウントでログインする必要があるため、他人のアカウントでスタンプを付与することはできない。

#### 4. QRコードを撮影してSNSで拡散
**対策**: トークン方式（将来の拡張）

```
https://liff.line.me/xxx?action=stamp&amount=10&token=RANDOM_STRING

- token を毎日変更する
- データベースで有効なトークンを管理
- 無効なトークンはエラー
```

**現時点での判断**: Phase 1では不要

理由:
1. 拡散されても1日1回制限で実害は限定的
2. そもそも歯科医院のQRを拡散する動機が薄い
3. 実装コストとリスクのバランスが悪い

### ✅ 推奨するセキュリティレベル

**Phase 1（最小限）**:
- ✅ LIFF認証チェック
- ✅ 1日1回制限
- ❌ トークン方式（不要）

**Phase 2（必要に応じて）**:
- ✅ 位置情報チェック（診療所の近くでのみ有効）
- ✅ 時間帯制限（診療時間内のみ）

**Phase 3（過剰対策）**:
- トークン方式（コスト高、メリット低）

```typescript
            type,
            amount: amountParam,
            location,

          setAmount(amountParam);
```

### 4. 自動付与API

**`app/api/stamps/auto/route.ts`**:

```typescript
import { NextResponse } from 'next/server';
import { supabase } from '@/lib/supabase';

export async function POST(request: Request) {
  try {
    const { userId, type, amount, location, campaign } = await request.json();

    console.log('[Auto Stamp] リクエスト:', { userId, type, amount, location, campaign });

    // 1日1回制限チェック
    const today = new Date().toISOString().split('T')[0];
    const startOfDay = `${today}T00:00:00.000Z`;
    const endOfDay = `${today}T23:59:59.999Z`;

    const { data: todayStamps, error: checkError } = await supabase
      .from('stamp_history')
      .select('*')
      .eq('user_id', userId)
      .eq('stamp_method', 'qr_scan')
      .gte('visit_date', startOfDay)
      .lte('visit_date', endOfDay);

    if (checkError) {
      console.error('[Auto Stamp] チェックエラー:', checkError);
      return NextResponse.json({
        success: false,
        message: 'チェックに失敗しました',
      }, { status: 500 });
    }

    if (todayStamps && todayStamps.length > 0) {
      console.log('[Auto Stamp] 本日すでに付与済み');
      return NextResponse.json({
        success: false,
        message: '今日はすでにスタンプを付与しました',
      }, { status: 400 });
    }

    // 現在のスタンプ数を取得
    const { data: profile, error: profileError } = await supabase
      .from('profiles')
      .select('stamp_count')
      .eq('id', userId)
      .single();

    if (profileError || !profile) {
      console.error('[Auto Stamp] プロフィール取得エラー:', profileError);
      return NextResponse.json({
        success: false,
        message: 'ユーザー情報の取得に失敗しました',
      }, { status: 500 });
    }

    const currentStampCount = profile.stamp_count ?? 0;
    const newStampCount = currentStampCount + amount;

    // スタンプ履歴を追加
    const qrCodeId = `AUTO-QR-${location || 'UNKNOWN'}-${Date.now()}`;
    const notes = campaign
      ? `自動付与（キャンペーン: ${campaign}）`
      : `自動付与（QR: ${location || 'unknown'}）`;

    const { error: insertError } = await supabase
      .from('stamp_history')
      .insert({
        user_id: userId,
        stamp_number: newStampCount,
        amount: amount,
        stamp_method: type,
        qr_code_id: qrCodeId,
        notes: notes,
        visit_date: new Date().toISOString(),
      });

    if (insertError) {
      console.error('[Auto Stamp] 挿入エラー:', insertError);
      return NextResponse.json({
        success: false,
        message: 'スタンプの付与に失敗しました',
      }, { status: 500 });
    }

    console.log('[Auto Stamp] 成功:', { newStampCount, amount });

    return NextResponse.json({
      success: true,
      stampCount: newStampCount,
      amount: amount,
    });
  } catch (error) {
    console.error('[Auto Stamp] エラー:', error);
    return NextResponse.json({
      success: false,
      message: 'サーバーエラーが発生しました',
    }, { status: 500 });
  }
}
```

## QRコード台座の設計

### 設置場所と種類

受付カウンターに**2種類のQRコード台座**を設置します。

| 台座種別 | 付与スタンプ数 | 対象 | URL |
|---------|--------------|------|-----|
| **プレミアム通院** | 10個 | 定期検診・治療 | `?action=stamp&type=qr&amount=10&location=premium` |
| **通常** | 5個 | フッ素塗布など | `?action=stamp&type=qr&amount=5&location=standard` |

### デザインのポイント

#### 1. サイズ感
- **台座サイズ**: 縦15cm × 横10cm × 奥行5cm（卓上型）
- **QRコードサイズ**: 7cm × 7cm（大きめ）
- **文字サイズ**: メインコピー 24pt、サブコピー 16pt

#### 2. 色分け
- **プレミアム通院（10個）**: ゴールド系の背景
  - 背景色: `#FFF8DC`（コーンシルク）
  - アクセント: `#FFD700`（ゴールド）

- **通常（5個）**: グリーン系の背景
  - 背景色: `#F0FFF0`（ハニーデュー）
  - アクセント: `#90EE90`（ライトグリーン）

#### 3. 視認性
- **目立つ配置**: 受付カウンターの患者側
- **2つ並べて設置**: 左右に並べて選択できるように
- **説明なしで分かる**: ビジュアルだけで理解できるデザイン

## 想定される課題と対策

### 課題1: Androidユーザーの対応

**問題**:
Androidの標準カメラアプリは、QRコード読み取り機能がない機種がある。

**対策**:
1. **Google Lens を推奨**
   - 「Google Lens でQRコードを読み取ってください」と案内
2. **アプリ内カメラを併用**
   - ポスターに「アプリ内のカメラからも読み込めます」と記載
3. **受付でサポート**
   - スタッフが操作を代行する手順を用意

**実装優先度**: 中

**理由**: iPhoneユーザーが多い（70%以上と推定）

---

### 課題2: LIFFアプリが起動しない

**問題**:
QRコードを読み取っても、LIFFアプリが開かずにブラウザで開いてしまう。

**対策**:
1. **URLスキームの確認**
   - `https://liff.line.me/...` が正しく設定されているか確認
2. **LINEアプリのインストール確認**
   - LINEアプリがインストールされていない場合、ブラウザで開く
   - 「LINEアプリをインストールしてください」というエラー画面を表示

**実装優先度**: 高

---

### 課題3: ネットワークエラー

**問題**:
診療所のWi-Fiが不安定で、API呼び出しが失敗する。

**対策**:
1. **リトライ機能**
   - 3回まで自動リトライ
2. **オフライン対応**
   - 一時的にローカルストレージに保存し、後で同期
3. **エラーメッセージ改善**
   - 「ネットワークエラーです。Wi-Fiを確認してください」と表示

**実装優先度**: 中

---

### 課題4: ユーザーが混乱する

**問題**:
「これでスタンプが押されたの？」と不安になるユーザーがいる。

**対策**:
1. **視覚的フィードバック**
   - スタンプが押される瞬間にアニメーション
   - 音（バイブレーション）で通知
2. **明確な完了画面**
   - 「スタンプを押しました！」と大きく表示
   - 現在のスタンプ数を表示
3. **ホーム画面への導線**
   - 「ホームに戻る」ボタンを大きく配置

**実装優先度**: 高

### 測定方法

#### 1. ログ収集

**実装箇所**: `app/api/stamps/auto/route.ts`

```typescript
// 成功ログ
await supabase.from('event_logs').insert({
  event_type: 'auto_stamp_success',

    amount,
```
