## 🚀 ローカル開発環境の起動

### 1. Next.js 開発サーバー起動

```bash
npm run dev
```

- **ポート**: `4000`
- **URL**: http://localhost:4000
- **説明**: Next.js の開発サーバーを起動します（ホットリロード有効）

### 2. ngrok でトンネル接続

**別のターミナルで実行**:

```bash
npx ngrok http 4000
```

- **ポート**: `4000` (Next.js サーバーと同じ)
- **説明**: ローカルサーバーを外部からアクセス可能にします（LIFF開発用）
- **出力例**:
  ```
  Forwarding  https://xxxx-yyyy-zzzz.ngrok-free.app -> http://localhost:4000
  ```
- **注意**: ngrok の URL を LIFF エンドポイント URL として LINE Developers に設定

### 📋 ローカル開発の完全起動手順

```bash
# ターミナル1: Next.js サーバー起動
npm run dev

# ターミナル2: ngrok トンネル起動
npx ngrok http 4000
```

**確認**:
1. ngrok の出力から `https://xxxx.ngrok-free.app` をコピー
2. LINE Developers > LIFF > エンドポイント URL に設定
3. LINE アプリから LIFF を開いて動作確認

## 🧪 テスト・検証コマンド

### データベーススキーマ全体確認

```bash
npx tsx test/check-all-schema.ts
```

**確認項目**:

2. stamp_history テーブル

4. family_stamp_totals ビュー

## 📦 ビルド・デプロイ

### ローカルビルド確認

```bash
npm run build
```

- **説明**: 本番ビルドを生成（エラーチェック）
- **注意**: ビルドエラーがないことを確認してからデプロイ

### バージョン番号の更新

```bash
npm run version:update
```

- **説明**: `package.json` のバージョンを更新
- **自動実行**: `npm run build` 実行時に自動実行される（prebuild スクリプト）

```bash
git add .
git commit -m "メッセージ"
git push origin main
```

- **説明**: main ブランチに push すると Vercel が自動デプロイ
- **確認**: Vercel Dashboard でデプロイ状況を確認

## 🛠️ その他の便利コマンド

### プロセス管理 (Windows)

```bash
# ポート 4000 を使用しているプロセスを確認
netstat -ano | findstr :4000

# プロセスを強制終了 (PID を確認してから実行)
taskkill /PID <プロセスID> /F

# Git Bashで taskkill が動作しない場合
cmd /c "taskkill /PID <プロセスID> /F"

# PowerShell でプロセス終了
powershell -Command "Stop-Process -Id <プロセスID> -Force"

# Node.js プロセス一覧
tasklist | findstr node

# 実行例: ポート4000を使用しているプロセス3500を終了
netstat -ano | findstr :4000
# → PID 3500 が表示された場合
cmd /c "taskkill /PID 3500 /F"
```

### 環境変数の確認

```bash
# .env.local が読み込まれているか確認
node -e "require('dotenv').config({path:'.env.local'}); console.log(process.env.NEXT_PUBLIC_SUPABASE_URL)"
```

### Git 操作

```bash
# 変更状況確認
git status

# 差分確認
git diff

# ブランチ確認
git branch

# コミット履歴
git log --oneline -10
```

## ⚠️ トラブルシューティング

### ポート 4000 が既に使用されている

```bash
# Windows
netstat -ano | findstr :4000
taskkill /PID <プロセスID> /F

# または別のポートで起動
npm run dev -- -p 4001
npx ngrok http 4001
```

### ngrok が起動しない

```bash
# ngrok がインストールされているか確認
npx ngrok version

# ngrok の認証（初回のみ）
npx ngrok authtoken <YOUR_AUTH_TOKEN>
```

```bash
# 環境変数を確認
cat .env.local

# テスト実行
```
