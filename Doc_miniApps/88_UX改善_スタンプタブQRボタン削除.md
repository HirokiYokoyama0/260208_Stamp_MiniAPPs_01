# UX改善：スタンプタブのQRスキャンボタン削除

**実装日**: 2026-03-28
**目的**: ボタンの重複を解消し、ユーザー動線を明確化

---

## 📋 背景

### 問題点

スタンプタブのQRスキャンボタンに以下の問題がありました：

1. **機能の重複**
   - 会員証タブとスタンプタブの両方に「来院スタンプを読み取る」ボタン
   - ユーザーがどちらを使えばいいか迷う

2. **異なるAPIを使用**
   - 会員証タブ: `/api/stamps/scan`（新しい、機能豊富）
   - スタンプタブ: `/api/stamps`（古い、シンプル）
   - 同じボタンなのに裏側の処理が違う → バグの温床

3. **購買インセンティブの制限問題**
   - スタンプタブの `/api/stamps` は全てのQRに1日1回制限
   - 購買インセンティブも制限されてしまう（本来は無制限）

4. **動線の混乱**
   - スタンプタブは「履歴を見る」場所
   - スタンプを取得するのは会員証タブが自然

---

## 🎯 実装内容

### スタンプタブのQRスキャンボタンを削除

[components/(adult)/AdultStampPage.tsx](components/(adult)/AdultStampPage.tsx)

```typescript
{/* QRスキャンボタン */}
<QRScanner
  onScan={handleStampScan}
  onError={(err) => console.error("QRスキャンエラー:", err)}

  className="w-full"

```

#### After（削除後）

```typescript
  <h2 className="mb-4 text-xs font-medium uppercase tracking-wider text-gray-400">
    スタンプ獲得履歴
  </h2>
  {/* ... */}
</section>
```

### 削除したコード

1. **QRScannerコンポーネント** - 削除
2. **handleStampScan関数** - 削除
3. **isScanning ステート** - 削除
4. **addStamp インポート** - 削除（未使用）
5. **不要なインポート** - 削除（Link, ArrowRight）
