## 📋 実施内容

### 無効化した機能

**「来院スタンプを読み取る」ボタン（アプリ内QRスキャナー）を一時的に非表示にしました。**

---

## 🎯 無効化の理由

### 問題の発生

診察券1019（谷田部寿恵様）と7766（天羽壱吉様）が、**カメラ用QRコード（15pt）をアプリ内スキャナーで読み込んだところ、10ポイントしか付与されない**という問題が発生しました。

### 根本原因

2種類のQRコードが混在していることが原因:

1. **カメラ用QR** (URL形式)
   - `https://liff.line.me/...?action=stamp&type=qr&amount=15&location=entrance`
   - カメラで読み込む → `/auto-stamp` ページに直接飛ぶ → 正しく動作 ✅
   - アプリ内スキャナーで読み込む → `/api/stamps` で処理 → バグ発生 ❌

2. **アプリ内スキャン用QR** (JSON形式)
   - `{"type":"premium","stamps":15}`
   - アプリ内スキャナーで読み込む → `/api/stamps` で処理 → 正しく動作 ✅

### なぜ無効化するのか

**ユーザーにとって「どちらのQRコードを使うべきか」が不明確**

- スタッフは「カメラ用QR」を提示
- ユーザーは「アプリ内スキャナーボタン」を押してスキャン
- 結果: 15pt → 10pt になる

**解決策**: アプリ内スキャン機能を一旦無効化し、**カメラで読み込む方法のみ**に統一。

## ✅ 修正後の動作

### ユーザーの操作フロー

**修正前**:
1. スタッフが「カメラ用QR」を提示
2. ユーザーが「来院スタンプを読み取る」ボタンをタップ
3. アプリ内スキャナーが起動
4. QRコードをスキャン → バグ発生（15pt → 10pt）

**修正後**:
1. スタッフが「カメラ用QR」を提示
2. ユーザーがスマホのカメラでQRコードを読み込む
3. `/auto-stamp` ページが開く
4. 正しく15ポイント付与 ✅

**「来院スタンプを読み取る」ボタンは表示されない**
