# バグ修正：引き渡し済み特典が期限切れと表示される問題

**修正日**: 2026-03-28
**バグ報告者**: クライアント（管理ダッシュボード確認時）
**深刻度**: 高（ユーザー体験を損なう）

## 🐛 問題の詳細

### 症状

引き渡し済み（`status = 'completed'`）の特典が、有効期限を過ぎると「有効期限切れ」と表示されてしまう。

**具体例：**
```
20個目の歯ブラシ
- 2026/03/27 取得
- 2026/03/27 引き渡し完了（completed）
- 有効期限: 2026/03/27 23:59:59

↓ 翌日（2026/03/28）

表示が「交換完了（緑）」から「有効期限切れ（グレー）」に変わってしまう ❌
```

### 期待される動作

引き渡し済み（`completed`）の特典は、有効期限が過ぎても「交換完了（緑バッジ）」を保持すべき。

---

## 🔍 原因分析

### 問題のコード

[components/(adult)/AdultRewardsPage.tsx:159](components/(adult)/AdultRewardsPage.tsx#L159)（修正前）

```typescript
// 有効期限チェック
const expirationCheck = latestExchange?.valid_until
  ? (() => {
      const now = new Date();
      const validUntil = new Date(latestExchange.valid_until);
      const isExpired = validUntil < now;  // ← 常にチェックしていた
      const diffMs = validUntil.getTime() - now.getTime();
      const diffDays = Math.ceil(diffMs / (1000 * 60 * 60 * 24));
      return {
        isExpired,
        daysRemaining: isExpired ? null : diffDays,
      };
    })()
  : { isExpired: false, daysRemaining: null };

const rewardWithStatus: MilestoneRewardWithStatus = {
  ...reward,
  isPending: latestExchange?.status === 'pending',
  isCompleted: latestExchange?.status === 'completed',
  isCancelled: latestExchange?.status === 'cancelled',
  // ❌ 問題：completed でも expirationCheck.isExpired = true になる
  isExpired: latestExchange?.status === 'expired' || expirationCheck.isExpired,
  // ...
};
```

### 問題の流れ

1. **3/27 10:00** - 10個目のスタンプ取得
   - `status = 'pending'`
   - `valid_until = '2026-03-27T23:59:59+09:00'`

2. **3/27 15:00** - 引き渡し完了
   - `status = 'completed'`
   - `valid_until = '2026-03-27T23:59:59+09:00'`（変更なし）

3. **3/28 10:00** - アプリを開く
   - `valid_until < now` → `true`（期限切れ）
   - `expirationCheck.isExpired = true`
   - **`isExpired = true`** になってしまう ❌

      return {
        isExpired,
        daysRemaining: isExpired ? null : diffDays,
      };

const rewardWithStatus: MilestoneRewardWithStatus = {

  isCompleted,

  // 1. status が 'expired' の場合
  // 2. status が 'pending' で valid_until を過ぎている場合

  isExpired: isExpiredStatus || (isPending && expirationCheck.isExpired),

### 変更のポイント

#### 1. ステータスを先に判定

```typescript
const isPending = latestExchange?.status === 'pending';
const isCompleted = latestExchange?.status === 'completed';
const isCancelled = latestExchange?.status === 'cancelled';
const isExpiredStatus = latestExchange?.status === 'expired';
```

#### 2. 有効期限チェックは `pending` の場合のみ

```typescript
// 修正前：常にチェック
const expirationCheck = latestExchange?.valid_until ? (...) : (...)

// 修正後：pending の場合のみチェック
const expirationCheck = isPending && latestExchange?.valid_until ? (...) : (...)
```

#### 3. 期限切れ判定ロジックを明確化

```typescript
// 修正前：すべてのステータスで期限チェック
isExpired: latestExchange?.status === 'expired' || expirationCheck.isExpired

// 修正後：pending の場合のみ期限チェック
isExpired: isExpiredStatus || (isPending && expirationCheck.isExpired)
```
