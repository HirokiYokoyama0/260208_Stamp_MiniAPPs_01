# iPhone問題：調査アプローチと統合分析

**作成日**: 2026-03-28
**最終更新**: 2026-03-28
**優先度**: 🔴 最高

## 📋 問題の状況

### 報告されている事象

**問題1**: 15スタンプQRが10ポイントしか付与されない（時々発生）
- 優良患者様用QR（15スタンプ想定）をスキャン → **10ポイントのみ付与**
- **別の携帯では15ポイント正常に付与された** ← 同じQRコードのはずが違う結果
- **同じiPhoneでも、うまくいく時とうまくいかない時がある** ← NEW!

**問題2**: 購買インセンティブQRが1度しかスキャンできない
- （この問題は問題1の調査結果を見てから判断）

### 重要な前提条件

| 項目 | 状態 |
|------|------|
| 開発者のAndroidスマホ | ✅ 正常動作確認済み |
| 15スタンプQR（優良患者様用） | ✅ 運用中 |
| 10スタンプQR（通常患者様用） | ❌ **未展開**（まだ1種類しか教えていない） |
| 5スタンプQR（購買インセンティブ） | ❌ 未展開または限定的 |
| コード実装（API・フロントエンド） | ✅ 正しい |
| イベントログ機能 | ✅ **実装完了** |

## 🎯 最も可能性が高い原因

### ⭐⭐⭐⭐⭐ 【最優先】院内に複数のQRコードが物理的に存在

**仮説**:
- 管理ダッシュボード側で15スタンプQRを**複数回**生成している
- 最初の生成時にペイロードが `{"type":"premium","stamps":10}` だった（ミス）
- 後で修正して再生成したQRコードは `{"type":"premium","stamps":15}` になった
- 院内のどこかに**古いQRコード**が残っている
- **ユーザーがどちらをスキャンするかで結果が変わる**

**「同じiPhoneでもうまくいく時とうまくいかない時がある」の説明**:
```
ケース1: 受付のQRコードをスキャン（古いQR）
  → {"type":"premium","stamps":10} → 10ポイント付与

ケース2: 待合室のQRコードをスキャン（新しいQR）
  → {"type":"premium","stamps":15} → 15ポイント付与

同じユーザーが異なる場所のQRコードをスキャン
→ 同じiPhoneでも結果が異なる！
```

**確率**: **95%** これが原因の可能性が非常に高い

---

## 🚀 調査アクション（優先順位順）

### 【最優先】Step 1: イベントログで `qr_raw_value` を確認

**所要時間**: 5分
**実施者**: 開発者（Supabase SQL Editorを使用）

#### SQL クエリ

```sql
-- 1. 最近のQRスキャンで、QRコードの生値を確認
SELECT
  created_at,
  user_id,
  metadata->>'qr_raw_value' as qr_raw,
  metadata->'qr_parsed'->>'type' as type,
  metadata->'qr_parsed'->>'stamps' as stamps_parsed,
  metadata->>'device_type' as device,
  metadata->'request_payload'->>'stamps' as stamps_requested
FROM event_logs
WHERE event_name = 'stamp_scan_api_request'
  AND created_at >= current_date - interval '7 days'
ORDER BY created_at DESC
LIMIT 50;
```

#### 確認ポイント

| パターン | 結果 | 意味 |
|---------|------|------|
| **A** | `qr_raw` に `"stamps":10` と `"stamps":15` の両方が混在 | 🎉 **原因特定！** 複数のQRコードが存在 |
| **B** | 全て `"stamps":10` | 管理ダッシュボード側でQRコード生成ミス |
| **C** | 全て `"stamps":15` | LIFF の scanCodeV2 の誤読の可能性（稀） |

#### 期待される結果

**パターンA** が最も可能性が高い：
```json
// スキャン1（ある場所のQRコード）
{"qr_raw": "{\"type\":\"premium\",\"stamps\":10}", "stamps_parsed": "10"}

// スキャン2（別の場所のQRコード）
{"qr_raw": "{\"type\":\"premium\",\"stamps\":15}", "stamps_parsed": "15"}
```

→ **複数のQRコードが院内に混在していることが証明される**

### 【高優先】Step 2: 管理ダッシュボード開発者に確認

**所要時間**: 10分
**実施者**: 管理ダッシュボード開発者に連絡

#### 質問事項

```
Q1: 15スタンプQRコードを生成した履歴を確認してください
    - 何回生成しましたか？
    - 初回生成: いつ？ペイロードは？
    - 再生成: したか？いつ？ペイロードは？

Q2: テスト用QRコードを生成したことはありますか？
    - ある → そのペイロードは？何個で設定しましたか？
    - ない → OK

Q3: 最新の15スタンプQRコードのペイロードを見せてください
    - 期待値: {"type":"premium","stamps":15}
    - 実際の値: ?

Q4: QRコード生成画面のスクリーンショットを見せてください
    - スタンプ個数の入力欄は？
    - 間違って10と入力した可能性はないか？
```

### 【中優先】Step 3: 院内の全QRコードをスキャンして確認

**所要時間**: 15分
**実施者**: クリニックスタッフ

#### 手順

1. **院内の全てのQRコード設置場所をリストアップ**
   - 受付
   - 待合室
   - 診察室
   - レジ
   - トイレ
   - その他

2. **スマホの標準カメラでスキャン**（LIFF アプリではなく）
   - iPhone の標準カメラでQRコードをスキャン
   - 自動的にブラウザが開き、URLが表示される
   - URLの中身を確認（ペイロードが見える）

3. **それぞれのペイロードを記録**
   - 受付: `{"type":"premium","stamps":15}` → ✅
   - 待合室: `{"type":"premium","stamps":10}` → ❌ **古いQR発見！**

4. **古いQRコードがあれば撤去**

---

### 【低優先】Step 4: LIFF の誤読を確認（Step 1-3で原因が特定できない場合のみ）

**可能性**: 5%未満

もし Step 1-3 で原因が特定できず、かつイベントログで全て `"stamps":15` の場合のみ実施。

#### 確認方法

1. **印刷品質の確認**
   - QRコードが鮮明に印刷されているか
   - 汚れ、折れ、光の反射などがないか

2. **QRコードのサイズを大きくする**
   - 現在のQRコードが小さすぎる可能性
   - 推奨: 5cm × 5cm 以上

3. **複数デバイスでテスト**
   - 同じQRコードを10回連続でスキャン
   - 毎回15ポイントが付与されるか確認

## 🎯 解決策

### 【パターンA】複数のQRコードが混在していた場合（可能性95%）

#### 対応手順

1. **古いQRコードを全て撤去**
   - Step 3 で発見した古いQRコードを撤去
   - スタッフに周知徹底

2. **正しいQRコードのみを設置**
   - 管理ダッシュボードで最新の15スタンプQRコードを確認
   - ペイロードが `{"type":"premium","stamps":15}` であることを確認
   - 院内の全箇所に正しいQRコードのみを設置

3. **再発防止策**
   - 管理ダッシュボード側でQRコード生成時に**確認画面**を追加
   - QRコード生成履歴を記録
   - 古いQRコードは明確に「無効」とマーク

4. **ユーザーへのアナウンス**
   - 「一部のQRコードが古いものでした。現在は修正済みです」
   - 「15スタンプが正しく付与されます」

#### 期待される効果

- **100%の確率で問題解決**
- ユーザーがどのQRコードをスキャンしても15ポイントが付与される

---

### 【パターンB】管理ダッシュボード側のQRコード生成ミス（可能性3%）

#### 対応手順

1. **管理ダッシュボード開発者に修正依頼**
   - QRコード生成ロジックを確認
   - 正しいペイロード `{"type":"premium","stamps":15}` で再生成

2. **院内の全QRコードを差し替え**

---

### 【パターンC】LIFF の scanCodeV2 の誤読（可能性2%）

#### 対応手順

1. **QRコードのサイズを大きくする**（5cm × 5cm 以上）
2. **印刷品質を上げる**（高解像度プリンター使用）
3. **設置環境を改善**（照明、角度など）
4. **LINE に問い合わせ**（LIFFのバグの可能性）
