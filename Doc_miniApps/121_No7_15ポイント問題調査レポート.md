## 🚨 原因特定: QRパース処理のバグ

### 証拠

スクリプト `scripts/analyze-1019-7766-environment.ts` の実行結果:

```
📊 スタンプ履歴分析:
   ⚠️ 問題の記録を発見:

      期待ポイント: 15
      実際の付与: 10
      メモ: ホームボタンスキャン (regular)
```

**これが証明すること**:
1. ユーザーは**正しく15ポイントのカメラ用QRコード**をスキャンした
2. QR URLには明確に `amount=15` が含まれている
3. しかし、システムは `regular` タイプ（10ポイント）として処理した
4. **QRコードの取り違えではない - これはバグである**

## 💡 バグの詳細

### 問題のコード: `app/api/stamps/route.ts` (Line 48-62)

```typescript
// QRコードペイロードのパース（タイプとスタンプ数を取得）
let qrType: 'regular' | 'premium' | 'purchase' = 'regular';
let stampAmount = STAMP_AMOUNTS.REGULAR_VISIT; // デフォルト: +10個
try {
  const qrPayload = JSON.parse(qrCodeId);
  if (qrPayload.type && ['regular', 'premium', 'purchase'].includes(qrPayload.type)) {
    qrType = qrPayload.type;
  }
  if (qrPayload.stamps && typeof qrPayload.stamps === 'number') {
    stampAmount = qrPayload.stamps;
  }
  console.log(`📱 [Stamps API] QRコードから読み取り: ${stampAmount}個, type: ${qrType}`);
} catch {
  // JSONパースエラー = 通常のQRコードID、デフォルト値を使用
}
```

### 何が起こったか

#### ステップ1: ユーザーがカメラ用QRコードをアプリ内スキャナーで読み込み

**QRコードの内容（カメラ用QR）**:
```
https://liff.line.me/2009075851-74EieWb4?action=stamp&type=qr&amount=15&location=entrance
```

**本来の用途**: カメラで読み込んで `/auto-stamp` ページに直接飛ぶ

**実際の動作**: アプリ内スキャナーで読み込んでしまった

#### ステップ2: アプリ内スキャナーがこのURLを `/api/stamps` に送信

**送信されたリクエスト**:
```json
{
  "userId": "Uac592c34b6d9d9dc84dbe9cc2c94f2c2",
  "qrCodeId": "https://liff.line.me/2009075851-74EieWb4?action=stamp&type=qr&amount=15&location=entrance"
}
```

#### ステップ3: `/api/stamps` がパース失敗

**期待されるペイロード（アプリ内スキャン用）**:
```json
{"type":"premium","stamps":15}
```

**実際に受け取ったqrCodeId**:
```
https://liff.line.me/2009075851-74EieWb4?action=stamp&type=qr&amount=15&location=entrance
```

**結果**:
- `JSON.parse(qrCodeId)` がエラー（URLはJSONではない）
- catch ブロックが実行
- デフォルト値に戻る:
  - `qrType = 'regular'`
  - `stampAmount = STAMP_AMOUNTS.REGULAR_VISIT` (10)

#### ステップ4: 10ポイントが付与される

```typescript
notes: `ホームボタンスキャン (${qrType})`, // "ホームボタンスキャン (regular)"
amount: stampAmount, // 10
```

## 🔧 修正方法

### 修正案: カメラ用QR URLのパース処理を追加

`app/api/stamps/route.ts` の Line 48-62 を以下のように修正:

```typescript
// QRコードペイロードのパース（タイプとスタンプ数を取得）
let qrType: 'regular' | 'premium' | 'purchase' = 'regular';
let stampAmount = STAMP_AMOUNTS.REGULAR_VISIT; // デフォルト: +10個

// 🆕 カメラ用QR URLの検出とパース処理
if (qrCodeId.startsWith('http')) {
  try {
    const url = new URL(qrCodeId);
    const action = url.searchParams.get('action');
    const type = url.searchParams.get('type');
    const amount = url.searchParams.get('amount');
    const location = url.searchParams.get('location');

    if (action === 'stamp' && amount) {
      stampAmount = parseInt(amount);

      // locationまたはamountからタイプを判定
      if (location === 'shop') {
        qrType = 'purchase';
      } else if (stampAmount === 15) {
        qrType = 'premium';
      } else if (stampAmount === 10) {
        qrType = 'regular';
      } else if (stampAmount === 5) {
        qrType = 'purchase';
      }

      console.log(`📱 [Stamps API] カメラ用QR URLから読み取り: ${stampAmount}個, type: ${qrType}`);
    }
  } catch (err) {
    console.error('❌ カメラ用QR URLのパースエラー:', err);
    // エラーならJSON形式を試す
  }
}

// JSONペイロード（アプリ内スキャン用）のパース
if (stampAmount === STAMP_AMOUNTS.REGULAR_VISIT && qrType === 'regular') {
  try {
    const qrPayload = JSON.parse(qrCodeId);
    if (qrPayload.type && ['regular', 'premium', 'purchase'].includes(qrPayload.type)) {
      qrType = qrPayload.type;
    }
    if (qrPayload.stamps && typeof qrPayload.stamps === 'number') {
      stampAmount = qrPayload.stamps;
    }
    console.log(`📱 [Stamps API] JSONペイロードから読み取り: ${stampAmount}個, type: ${qrType}`);
  } catch {
    // JSONパースエラー = 通常のQRコードID、デフォルト値を使用
    console.log(`📱 [Stamps API] デフォルト値を使用: ${stampAmount}個, type: ${qrType}`);
  }
}
```

### 修正のポイント

1. **カメラ用QR URL形式を検出**: `qrCodeId.startsWith('http')`
2. **URLパラメータをパース**: `new URL(qrCodeId)` でパース
3. **`amount`パラメータから正しいスタンプ数を取得**: `parseInt(amount)`
4. **amountに基づいてタイプを判定**:
   - 15個 → `premium`
   - 10個 → `regular`
   - 5個 → `purchase`
5. **既存のJSONパース処理はそのまま維持**: アプリ内スキャン用QRとの互換性

## 📝 まとめ

### 問題の真相

**QRパース処理のバグ** が原因。

1. ユーザーが**カメラ用QRコード（15pt）**をアプリ内スキャナーで読み込み
2. `/api/stamps` がカメラ用QR URL形式に対応していなかった
3. JSONパースエラー → デフォルト値（10pt, regular）にフォールバック
4. 15ポイントのつもりが10ポイントしか付与されない

### 修正内容

- `app/api/stamps/route.ts` の QRパース処理を修正
- カメラ用QR URL形式（`https://liff.line.me/...?amount=15`）に対応
- URLパラメータから正しいスタンプ数とタイプを取得
