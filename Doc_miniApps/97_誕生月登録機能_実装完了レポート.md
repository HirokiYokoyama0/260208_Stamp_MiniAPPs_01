## 1. 実装概要

管理ダッシュボード開発者の要望に基づき、ユーザーが誕生月を登録できる機能を実装しました。これにより、管理ダッシュボードから誕生月ユーザーを抽出し、お誕生日クーポンの自動配信が可能になります。

  if (!profile?.userId || birthMonth !== null) return;

  const today = new Date().toISOString().split('T')[0];
  const lastPrompt = localStorage.getItem('birthMonthPrompt_lastShown');

  if (lastPrompt === today) return;

  if (lastSkip === today) return;

    }
  }}

```typescript
export function BirthMonthPromptModal({
  isOpen,
  onClose,
  userId,
  onSuccess,
}: BirthMonthPromptModalProps) {
  const [birthMonth, setBirthMonth] = useState<number | null>(null);
  const [isSaving, setIsSaving] = useState(false);

  const handleSave = async () => {
    if (!birthMonth) {
      alert('誕生月を選択してください');
      return;
    }

    setIsSaving(true);
    const { error } = await supabase
      .from('profiles')
      .update({
        birth_month: birthMonth,
        updated_at: new Date().toISOString(),
      })
      .eq('id', userId);

    if (!error) {
      alert('誕生月を登録しました！\nお誕生月にクーポンをお届けします🎁');
      onSuccess?.();
      onClose();
    }
    setIsSaving(false);
  };

  const handleSkip = () => {
    const today = new Date().toISOString().split('T')[0];
    localStorage.setItem('birthMonthPrompt_lastSkip', today);
    onClose();
  };

  // ... モーダルUI
}
```

## 3. データフロー

### 3.1 新規ユーザー（オンボーディング経由）

```
オンボーディング画面
  ↓ 誕生月選択（任意）
/api/users/setup-role
  ↓ POST { birthMonth: 3 }
profiles.birth_month = 3
  ↓
ホーム画面でポップアップ表示されない
```

### 3.2 既存ユーザー（誕生月未登録）

```
ホーム画面起動
  ↓ birth_month IS NULL を検出
  ↓ localStorage チェック（今日表示済みか？）
  ↓ 2秒待機
BirthMonthPromptModal 表示
  ↓ ユーザーが誕生月を選択
  ↓ 「登録する」クリック
Supabase UPDATE
  ↓
profiles.birth_month = 3
  ↓
fetchUserData() 再実行
  ↓
次回起動時にポップアップ表示されない
```

### 3.3 既存ユーザー（設定画面経由）

```
設定画面
  ↓ プロフィール編集モード
  ↓ 誕生月選択
  ↓ 「保存」クリック
Supabase UPDATE
  ↓
profiles.birth_month = 5
```

## 4. localStorage 管理

### 4.1 使用キー

| キー | 保存タイミング | 用途 |
|------|--------------|------|
| `birthMonthPrompt_lastShown` | モーダル表示時 | 1日1回のみ表示制御 |
| `birthMonthPrompt_lastSkip` | 「後で登録」クリック時 | スキップした日は再表示しない |

### 4.2 表示ロジック

モーダルを表示する条件（全て満たす必要がある）:
1. ユーザーがログイン済み（`profile?.userId` が存在）
2. 誕生月が未登録（`birthMonth === null`）
3. 今日まだ表示していない（`birthMonthPrompt_lastShown !== 今日`）
4. 今日スキップしていない（`birthMonthPrompt_lastSkip !== 今日`）

## 7. 管理ダッシュボード側の対応

管理ダッシュボード開発者が実施すべき内容:

### 7.1 誕生月ユーザーの抽出

```sql
-- 今月が誕生月のユーザーを抽出
SELECT id, display_name, real_name, birth_month
FROM profiles
WHERE birth_month = EXTRACT(MONTH FROM CURRENT_DATE)
  AND birth_month IS NOT NULL;
```

### 7.2 クーポン配信

- 抽出したユーザーに LINE メッセージ API でクーポンを送信
- 月初に自動実行するバッチ処理を実装

## 8. 今後の拡張案

### 8.1 誕生日（日付）の登録

現在は「誕生月」のみだが、将来的に「誕生日（日付）」も登録できるように拡張可能:

```sql
ALTER TABLE profiles ADD COLUMN birth_date DATE;
```

### 8.2 登録率の向上施策

- オンボーディング時に誕生月を「必須」にする（現在は任意）
- ポップアップの表示頻度を変更（現在は1日1回）
- クーポン特典の訴求を強化

### 8.3 統計情報

管理ダッシュボードに誕生月登録率を表示:

```sql
SELECT
  COUNT(*) AS total_users,
  COUNT(birth_month) AS registered_users,
  ROUND(COUNT(birth_month)::NUMERIC / COUNT(*) * 100, 2) AS registration_rate
FROM profiles;
```
