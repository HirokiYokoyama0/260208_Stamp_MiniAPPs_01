# 緊急修正：「本日のQRスキャンを削除」機能のタイムゾーン不整合

**作成日**: 2026-03-28
**優先度**: 🔴 緊急（本番環境で機能不全）
**ステータス**: ✅ 修正完了

## 📋 報告された問題

スタッフ操作の「本日のQRスキャンを削除」機能が効かなくなっている。

**具体的な症状**:
- スタンプ数は減っている（削除自体は成功）
- しかし、もう一度同じ日にQRをスキャンできない（1日1回制限が効いたまま）

## 🔍 根本原因

### タイムゾーンの不整合

スキャンAPIと削除APIで**異なるタイムゾーン**を使用していた：

#### スキャンAPI（正しい実装）
[app/api/stamps/scan/route.ts:112-115](app/api/stamps/scan/route.ts#L112-L115)

```typescript
// JST（日本時間 = UTC+9）で計算
const nowJST = new Date(new Date().getTime() + 9 * 60 * 60 * 1000);
const todayJST = nowJST.toISOString().split("T")[0];
const startOfDayUTC = `${todayJST}T00:00:00+09:00`; // JSTの0時
const endOfDayUTC = `${todayJST}T23:59:59.999+09:00`; // JSTの23:59:59

// 1日1回制限チェック
.gte("visit_date", startOfDayUTC)
.lt("visit_date", endOfDayUTC)
```

#### 削除API（間違った実装）
[app/api/stamps/scan/delete-today/route.ts:50-53](app/api/stamps/scan/delete-today/route.ts#L50-L53) - 修正前

```typescript
// UTC で計算（サーバーのローカル時刻に依存）
const now = new Date();
const today = now.toISOString().split("T")[0];
const startOfDay = `${today}T00:00:00.000Z`;  // UTCの0時
const endOfDay = `${today}T23:59:59.999Z`;    // UTCの23:59:59

// 削除対象検索
.gte("visit_date", startOfDay)
.lte("visit_date", endOfDay)  // ← .lt() ではなく .lte() も問題
```

### 問題発生のメカニズム

**例**: 日本時間 2026-03-28 10:00 AM にスキャン

1. **スキャン時**:
   - `visit_date = 2026-03-28T01:00:00Z`（UTC）として保存
   - 1日1回チェック範囲: JST基準で `2026-03-27T15:00:00Z` ～ `2026-03-28T14:59:59.999Z`（UTC換算）
   - ✅ 範囲内にレコードなしなのでスキャン成功

2. **削除時**:
   - サーバーがUTCで動作していると仮定
   - 削除範囲: `2026-03-28T00:00:00.000Z` ～ `2026-03-28T23:59:59.999Z`
   - `2026-03-28T01:00:00Z` は範囲内
   - ✅ 削除成功

3. **再スキャン時** （同じ日の午後）:
   - 1日1回チェック範囲: JST基準で `2026-03-27T15:00:00Z` ～ `2026-03-28T14:59:59.999Z`
   - しかし、サーバー時刻が UTC の午前中の場合、削除APIは「今日」を UTC 基準で判定
   - **削除APIとスキャンAPIで「今日」の定義が異なる** → 削除されたはずのレコードが再度見つかる可能性

   または、より単純に：
   - 削除範囲とチェック範囲が微妙にずれているため、削除されなかったレコードが残る

### より正確な問題パターン

実際には、サーバーのローカル時刻とJSTのずれにより、以下のような問題が発生：

- **サーバーがUTC午前0時 ～ 午前8時59分の場合**（JST午前9時～午後5時59分）:
  - 削除API: UTC基準で「今日」= まだ前日のJST
  - スキャンAPI: JST基準で「今日」
  - → 削除APIは前日のJSTレコードを削除、スキャンAPIは当日のJSTをチェック
  - → **完全にずれる**

## 🔧 修正内容

### 修正1: タイムゾーン計算の統一

[app/api/stamps/scan/delete-today/route.ts:49-55](app/api/stamps/scan/delete-today/route.ts#L49-L55)

```typescript
// 修正前（UTC基準）
const now = new Date();
const today = now.toISOString().split("T")[0];
const startOfDay = `${today}T00:00:00.000Z`;
const endOfDay = `${today}T23:59:59.999Z`;

// 修正後（JST基準 - スキャンAPIと同じ）
const nowJST = new Date(new Date().getTime() + 9 * 60 * 60 * 1000);
const todayJST = nowJST.toISOString().split("T")[0];
const startOfDay = `${todayJST}T00:00:00+09:00`; // JSTの0時をISO形式で
const endOfDay = `${todayJST}T23:59:59.999+09:00`; // JSTの23:59:59をISO形式で
```

### 修正2: クエリ演算子の統一

[app/api/stamps/scan/delete-today/route.ts:63-64](app/api/stamps/scan/delete-today/route.ts#L63-L64)

```typescript
// 修正前
.lte("visit_date", endOfDay);  // 以下（<=）

// 修正後（スキャンAPIと同じ）
.lt("visit_date", endOfDay);   // 未満（<）
```

これにより、スキャンAPIと削除APIで**完全に同じ日付範囲**を使用するようになりました。

## 📝 技術的補足

### なぜサーバーのローカル時刻に依存してはいけないか

```javascript
// サーバーがどこにあるかで結果が変わる
const now = new Date();  // ❌ サーバーのローカル時刻
const today = now.toISOString().split("T")[0];  // ❌ サーバーのローカル日付

// Vercel, AWS Lambda などのクラウドサービス → UTC
// 開発マシン（日本） → JST
// → 同じコードでも異なる結果！
```

### 正しい実装パターン

```javascript
// ユーザーのタイムゾーン（JST = UTC+9）で明示的に計算
const nowJST = new Date(new Date().getTime() + 9 * 60 * 60 * 1000);  // ✅
const todayJST = nowJST.toISOString().split("T")[0];  // ✅
const startOfDayUTC = `${todayJST}T00:00:00+09:00`;  // ✅ タイムゾーン情報を含める
```

### PostgreSQL のタイムスタンプ比較

```sql
-- visit_date は TIMESTAMPTZ 型（タイムゾーン付きタイムスタンプ）
-- ISO 8601 形式で +09:00 を付けると、PostgreSQL が自動的にUTCに変換して比較

-- 例: JSTの0時 = UTCの前日15時
'2026-03-28T00:00:00+09:00' = '2026-03-27T15:00:00Z'  -- ✅ 同じ時刻

-- したがって、スキャンAPI と 削除API で同じ範囲指定をすれば、
-- PostgreSQL側で正しく比較される
```

## 🎯 今後の対応

### 推奨事項
このパターン（JST基準の日付計算）を他のAPIでも統一することを推奨します：

1. **確認が必要なファイル**:
   ```bash
   # 日付範囲を扱う全APIを検索
   grep -r "toISOString" app/api/
   grep -r "new Date()" app/api/
   ```

2. **統一すべき箇所**:
   - 1日1回制限チェック
   - 日付範囲での検索
   - 当日のデータ削除

3. **ヘルパー関数の作成も検討**:
   ```typescript
   // lib/date-utils.ts
   export function getJSTDate() {
     return new Date(new Date().getTime() + 9 * 60 * 60 * 1000);
   }

   export function getJSTDateRange(date: Date) {
     const dateStr = date.toISOString().split("T")[0];
     return {
       start: `${dateStr}T00:00:00+09:00`,
       end: `${dateStr}T23:59:59.999+09:00`,
     };
   }
   ```
