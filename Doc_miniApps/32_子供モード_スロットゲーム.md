# スロットゲーム ルール仕様書

**作成日:** 2026-02-28
**最終更新:** 2026-04-11
**対象:** スロットゲーム実装担当者・デザイナー

---

## 🎰 ゲーム概要

子供用モードのスロットゲーム。3つのリールをタップして止め、絵柄の組み合わせでスタンプを獲得する。

**アクセス条件:**
- **来院時のみ**プレイ可能（院内QRコードの読み取りが必要）
- キッズモードのホーム画面に「📷 QRをよんで ゲームかいほう！」ボタンが表示される
- QR読み取り後、「🎰 ゲームで あそぶ！」ボタンが出現
- URL: `/slot`

---

## 📷 スロット解放の仕組み（来院トリガー）

### なぜ来院トリガーにするのか？

| 理由 | 説明 |
|------|------|
| 来院インセンティブ | 自宅で遊べると来院動機にならない |
| 特別感 | 「びょういんに きたら あそべる」というワクワク感 |
| 不正防止 | QRは院内設置のため院外では読めない |

### QRコード仕様

**ペイロード型QR（アプリ内スキャン専用）:**

```json
{"type":"slot-unlock","target":"family"}
```

| フィールド | 値 | 説明 |
|-----------|-----|------|
| `type` | `"slot-unlock"` | スロット解放専用（スタンプ付与なし） |
| `target` | `"family"` | 将来の拡張用（現在は表示中の子供を解放） |

> 既存のスタンプ付与QR（`premium`/`regular`/`purchase`）とは別の専用QR。
> 院内の受付または待合室に1枚設置する。

### 解放フロー

```
【キッズ画面（スロット未解放時）】
  「📷 QRをよんで ゲームかいほう！」ボタンを表示
    ↓ タップ
  liff.scanCodeV2() でカメラ起動
    ↓ 院内QRを読み取り
  ペイロード検証: type === "slot-unlock" ?
    ↓ YES
  POST /api/slot/unlock { userId: 表示中の子供ID }
    ↓
  event_logs に記録（event_name: "slot_unlock"）※stamp_historyは使わない
    ↓
  「🎰 ゲーム かいほう！」完了メッセージ表示
    ↓
  「🎰 ゲームで あそぶ！」ボタンが出現

【スマホなし子供が複数いる場合】
  子供Aの画面でQR読み取り → 子供Aのスロット解放
  子供Bに切り替え → 子供Bの画面でQR読み取り → 子供Bのスロット解放
  ※ 子供ごとに1回ずつスキャンが必要
```

### 解放判定ロジック

```typescript
// KidsHome.tsx での判定
// ※ stamp_historyではなくevent_logsを使用（スタンプ付与しないため）
event_logs WHERE user_id = 表示中の子供ID
  AND event_name = 'slot_unlock'
  AND created_at >= 今日 0:00 (JST)
→ 1件以上あればスロットボタン表示
```

> **設計方針:** スロット解放はスタンプ付与ではないため、`stamp_history` ではなく `event_logs` に記録する。`stamp_history` はスタンプ増減のある操作のみに使用する。

### 画面状態の遷移

| 状態 | スロットボタン | 表示内容 |
|------|-------------|---------|
| **来院前（未解放）** | 非表示 | 「📷 QRをよんで ゲームかいほう！」ボタン |
| **解放済み（残り2回）** | 表示 | 「🎰 ゲームで あそぶ！」ボタン |
| **2回遊んだ後** | グレーアウト | 「きょうの スロットは おわりだよ！⭐○こ ゲット！」 |

### エッジケース

| ケース | 対応 |
|--------|------|
| 同じ子供が2回スキャン | 「もう かいほうずみ だよ！」と表示（重複記録しない） |
| 不正なQRコード | 「このQRコードは つかえないよ」と表示 |
| liff.scanCodeV2 非対応環境 | フォールバックメッセージ表示 |
| 院外でQR撮影 | QRは院内設置のみなので低リスク |

## 🎮 2回チャレンジルール（ベストスコア採用）

1日2回遊べて、**得点が高かった方のスタンプが付与される**。

| 回 | スタンプ付与 | 画面表示 |
|----|-----------|---------|
| **1回目** | まだ付与しない | 「もう1かい あそべるよ！いいほうが スタンプになるよ！」 |
| **2回目** | **高い方を付与** | 「いいほう → ⭐○こ ゲット！」 |

### なぜベストスコア採用にするのか？

| 理由 | 説明 |
|------|------|
| 2回遊べる楽しさ | 1回で終わらず、もう1回チャンスがある |
| 納得感 | 結果が悪くても「もう1回ある」と前向きになれる |
| 公平性 | 操作ミスがあっても2回目で挽回できる |

## ⚙️ ゲーム操作フロー

```
【1回目】
1. 「🎰 1かいめ / 2かい」と表示
2. 「まわす！」ボタンをタップ → リール回転開始
3. リールをタップ or「とめる！」ボタンで停止
4. 結果表示（あたり/はずれ）
   → スコアを記録（まだスタンプ付与しない）
   → 「もう1かい あそべるよ！いいほうが スタンプになるよ！」

【2回目】
5. 「🎰 2かいめ / 2かい（1かいめ: ⭐○）」と表示
6. 「まわす！」ボタンをタップ → リール回転開始
7. リールをタップ or「とめる！」ボタンで停止
8. 結果表示
   → 2回のスコアを比較し、高い方のスタンプを付与
   → あたり: 紙吹雪エフェクト + サウンド

【終了】
9. 「1かいめ: ⭐○  2かいめ: ⭐○」両方のスコア表示
10. 「いいほう → ⭐○こ ゲット！」
11. ケア記録への遷移ボタン表示
```

## 📝 実装上の注意

- スタンプ付与のAPI呼び出しは `POST /api/stamps/slot` を使用
- スロット解放のAPI呼び出しは `POST /api/slot/unlock` を使用
- **1日2回まで**プレイ可能（`DAILY_LIMIT = 2`、localStorageで日付管理、翌日リセット）
- **2回遊んで高い方のスコアをスタンプとして付与（ベストスコア採用）**
- 1回目はスコア記録のみ、2回目完了時にAPI呼び出しで高い方を付与
- 2回終了後 →「きょうの スロットは おわりだよ！」メッセージ表示 + 画面グレーアウト
- 終了画面に「きょうは ⭐○こ ゲット！」と今日の獲得スタンプ合計を表示
- ケア記録への遷移ボタンを表示
- 判定ロジック: `WINNING_PATTERNS[reel0 + reel1 + reel2]` のキー完全一致
- **アプリ内スキャン（`liff.scanCodeV2()`）はキッズ画面専用**（大人画面では使用しない）

---

## 🔓 隠し機能

- グレーアウト状態で **1秒以内に3回タップ** すると回数制限を解除（開発・テスト用）
- 解除後は再度スロットをプレイ可能
