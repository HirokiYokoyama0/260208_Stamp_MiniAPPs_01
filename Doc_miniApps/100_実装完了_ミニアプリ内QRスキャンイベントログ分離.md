# ミニアプリ内QRスキャンイベントログの分離実装

**作成日**: 2026-03-28
**実装者**: Claude Code
**バージョン**: 0.0.0-dev

---

## 📋 概要

カメラQRスキャンとミニアプリ内QRスキャンボタンのイベントログを区別するため、別々のイベント名で記録するように実装しました。

### 問題点

- ミニアプリ内の「来院スタンプを読み取る」ボタンのイベントログが管理ダッシュボードで確認できない
- カメラQRスキャンとミニアプリ内スキャンが同じイベント名 `stamp_scan_success` で記録されていた
- どちらのスキャン方法が使用されたか判別できない

---

## ✅ 実装内容

### 1. イベント名の分離

| スキャン方法 | イベント名 | 説明 |
|-------------|-----------|------|
| カメラQR直接起動 | `stamp_scan_success` | カメラでQRコードをスキャンしてLIFFを直接起動 |
| ミニアプリ内スキャン | `stamp_scan_success_in_app` | ミニアプリ内の「来院スタンプを読み取る」ボタン |

### 2. 変更ファイル

#### [lib/analytics.ts](../lib/analytics.ts)

**変更内容**: `logStampScanSuccess` 関数に `scanMethod` パラメータを追加

```typescript
export const logStampScanSuccess = (params: {
  stampsAdded: number;
  type: 'regular' | 'premium' | 'purchase';
  userId?: string;
  scanMethod?: 'camera' | 'in_app';  // 🆕 スキャン方法を追加
  currentStampCount?: number;
  newStampCount?: number;
  stampHistoryId?: string;
  milestonesGranted?: Array<{ milestone: number; rewardType: string; exchangeId: string }>;
}) => {
  // スキャン方法に応じてイベント名を分ける
  const eventName = params.scanMethod === 'in_app'
    ? 'stamp_scan_success_in_app'  // ミニアプリ内スキャン
    : 'stamp_scan_success';         // カメラQR直接起動

  return logEvent({
    eventName,
    userId: params.userId,
    metadata: {
      stamps_added: params.stampsAdded,
      type: params.type,
      scan_method: params.scanMethod || 'camera',  // デフォルトはカメラ
      current_stamp_count: params.currentStampCount,
      new_stamp_count: params.newStampCount,
      stamp_history_id: params.stampHistoryId,
      milestones_granted: params.milestonesGranted,
    },
  });
};
```

## 🔍 調査用SQLクエリ

### 1. カメラQRスキャンとミニアプリ内スキャンの比較

```sql
-- スキャン方法別の集計
SELECT
  event_name,
  metadata->>'scan_method' AS scan_method,
  COUNT(*) AS event_count,
  AVG((metadata->>'stamps_added')::int) AS avg_stamps_added
FROM event_logs
WHERE event_name IN ('stamp_scan_success', 'stamp_scan_success_in_app')
  AND created_at >= CURRENT_DATE - INTERVAL '7 days'
GROUP BY event_name, metadata->>'scan_method'
ORDER BY event_name, scan_method;
```

**期待される結果**:
```
event_name                    | scan_method | event_count | avg_stamps_added
------------------------------|-------------|-------------|-----------------
stamp_scan_success            | camera      | 45          | 11.5
stamp_scan_success_in_app     | in_app      | 23          | 10.8
```

### 2. ミニアプリ内スキャンの詳細ログ

```sql
-- ミニアプリ内スキャンのみを抽出
SELECT
  created_at,
  user_id,
  metadata->>'type' AS stamp_type,
  metadata->>'stamps_added' AS stamps_added,
  metadata->>'current_stamp_count' AS before_count,
  metadata->>'new_stamp_count' AS after_count,
  metadata->>'milestones_granted' AS milestones
FROM event_logs
WHERE event_name = 'stamp_scan_success_in_app'
ORDER BY created_at DESC
LIMIT 20;
```

### 3. スキャン方法別のスタンプタイプ分布

```sql
-- どのスキャン方法でどのスタンプタイプが使われているか
SELECT
  metadata->>'scan_method' AS scan_method,
  metadata->>'type' AS stamp_type,
  metadata->>'stamps_added' AS stamps_added,
  COUNT(*) AS count
FROM event_logs
WHERE event_name IN ('stamp_scan_success', 'stamp_scan_success_in_app')
  AND created_at >= CURRENT_DATE - INTERVAL '30 days'
GROUP BY scan_method, stamp_type, stamps_added
ORDER BY scan_method, stamp_type, stamps_added;
```

**期待される結果**:
```
scan_method | stamp_type | stamps_added | count
------------|------------|--------------|------
camera      | premium    | 15           | 12
camera      | regular    | 10           | 28
in_app      | premium    | 15           | 8
in_app      | regular    | 10           | 15
```

## 🧪 テスト方法

### カメラQRスキャンのテスト

1. **QRコードを印刷またはスマホで表示**
2. **カメラアプリでスキャン**
   - iPhoneのカメラアプリ
   - AndroidのGoogle Lens等
3. **LIFFアプリが起動してスタンプ付与**
4. **Vercelログで確認**:
   ```
   ✅ [Analytics] Event log inserted: {
     eventName: 'stamp_scan_success',
     userId: 'U12345...',
     hasMetadata: true
   }
   ```
5. **管理ダッシュボードで確認**:
   - イベント名: `stamp_scan_success`
   - metadata.scan_method: `camera`

1. **LIFFアプリを開く**
2. **「来院スタンプを読み取る」ボタンをタップ**

4. **スタンプ付与後、Vercelログで確認**:
   ```
   🔍 [API] スタンプスキャン成功ログを送信: {
     stampsAdded: 10,
     type: 'regular',
     userId: 'U12345...',
     currentStampCount: 135,
   ```

5. **管理ダッシュボードで確認**:
   - イベント名: `stamp_scan_success_in_app`

## 🔧 トラブルシューティング

### イベントログが記録されない場合

#### 1. Supabase RLSポリシーの確認

```sql
-- event_logsテーブルのINSERTポリシー確認
SELECT * FROM pg_policies WHERE tablename = 'event_logs';
```

**期待される設定**:
- INSERT: ANONロールに対して許可されている必要あり
- もしくは、INSERT時にサービスロールキーを使用

#### 2. 環境変数の確認

```bash
# .env.local
NEXT_PUBLIC_SUPABASE_URL=https://xxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

#### 3. ブラウザコンソールでのエラー確認

```javascript
// 開発環境でのみ確認可能
// F12 → Console タブ
// エラーログの確認:
// ❌ [Analytics] Event log insertion failed: ...
```

#### 4. Vercelログでのエラー確認

Vercel Dashboard → Logs → Functions で以下を確認:
- `❌ [API] イベントログ記録エラー`
- `⚠️ [API] イベントログ記録失敗`
