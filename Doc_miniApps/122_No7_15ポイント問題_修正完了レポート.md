## 📋 問題の概要

### 発生した問題

診察券1019（谷田部寿恵様）と7766（天羽壱吉様）が**15ポイントのカメラ用QRコードをアプリ内スキャナーで読み込んだところ、10ポイントしか付与されなかった**。

### 原因

`app/api/stamps/route.ts` のQRパース処理が**カメラ用QR URL形式に対応していなかった**。

**問題のコード（修正前）**:
```typescript
// QRコードペイロードのパース（タイプとスタンプ数を取得）
let qrType: 'regular' | 'premium' | 'purchase' = 'regular';
let stampAmount = STAMP_AMOUNTS.REGULAR_VISIT; // デフォルト: +10個
try {
  const qrPayload = JSON.parse(qrCodeId); // ← URL形式だとエラー
  // ... パース処理
} catch {
  // ← デフォルト値（10pt, regular）にフォールバック
}
```

**何が起こったか**:
1. カメラ用QR URL: `https://liff.line.me/...?amount=15` をアプリ内スキャナーで読み込み
2. URLはJSON形式ではないので `JSON.parse()` がエラー
3. catch ブロックで **デフォルト値（10pt, regular）** にフォールバック
4. 結果: 15ポイントが10ポイントになる

## ✅ 修正内容

### 修正したファイル

- [app/api/stamps/route.ts](app/api/stamps/route.ts) (Line 48-98)

### 修正のポイント

1. **カメラ用QR URL形式を検出**: `qrCodeId.startsWith('http')`
2. **URLパラメータをパース**: `new URL(qrCodeId)` で `amount` パラメータを取得
3. **amountに基づいてタイプを判定**:
   - 15個 → `premium`
   - 10個 → `regular`
   - 5個 → `purchase`
4. **既存のJSONパース処理を維持**: アプリ内スキャン用QRとの互換性

### 修正後のコード

```typescript
// QRコードペイロードのパース（タイプとスタンプ数を取得）
let qrType: 'regular' | 'premium' | 'purchase' = 'regular';
let stampAmount = STAMP_AMOUNTS.REGULAR_VISIT; // デフォルト: +10個

// 🆕 カメラ用QR URLの検出とパース処理
if (qrCodeId.startsWith('http')) {
  try {
    const url = new URL(qrCodeId);
    const action = url.searchParams.get('action');
    const amount = url.searchParams.get('amount');
    const location = url.searchParams.get('location');

    if (action === 'stamp' && amount) {
      stampAmount = parseInt(amount);

      // locationまたはamountからタイプを判定
      if (location === 'shop') {
        qrType = 'purchase';
      } else if (stampAmount === 15) {
        qrType = 'premium';
      } else if (stampAmount === 10) {
        qrType = 'regular';
      } else if (stampAmount === 5) {
        qrType = 'purchase';
      }

      console.log(`📱 [Stamps API] カメラ用QR URLから読み取り: ${stampAmount}個, type: ${qrType}`);
    }
  } catch (err) {
    console.error('❌ カメラ用QR URLのパースエラー:', err);
  }
}

// JSONペイロード（アプリ内スキャン用）のパース
if (stampAmount === STAMP_AMOUNTS.REGULAR_VISIT && qrType === 'regular') {
  try {
    const qrPayload = JSON.parse(qrCodeId);
    if (qrPayload.type && ['regular', 'premium', 'purchase'].includes(qrPayload.type)) {
      qrType = qrPayload.type;
    }
    if (qrPayload.stamps && typeof qrPayload.stamps === 'number') {
      stampAmount = qrPayload.stamps;
    }
    console.log(`📱 [Stamps API] JSONペイロードから読み取り: ${stampAmount}個, type: ${qrType}`);
  } catch {
    console.log(`📱 [Stamps API] デフォルト値を使用: ${stampAmount}個, type: ${qrType}`);
  }
}
```

## 🔍 根本原因分析

### なぜこのバグが発生したか

1. **QRコードが2種類存在**:
   - カメラ用QR: URL形式（`https://liff.line.me/...?amount=15`）
   - アプリ内スキャン用QR: JSON形式（`{"type":"premium","stamps":15}`）

2. **APIはJSON形式のみ想定**:
   - `/api/stamps` はアプリ内スキャン用QR（JSON形式）のみを想定
   - カメラ用QRはカメラで読み込んで `/auto-stamp` ページに直接飛ぶ想定

3. **ユーザーがカメラ用QRをアプリ内スキャナーで読んだ**:
   - 本来の用途ではない
   - しかし、ユーザーにとってはどちらも「QRコード」
   - システムは柔軟に対応すべき

### 今後の対策

1. **✅ 実施済み**: `/api/stamps` でカメラ用QR URL形式に対応
2. **今後検討**: ユーザーに対して「どちらのQRコードを使っても同じように動作する」ことを周知
