## ❌ 問題箇所1: キッズスタンプページの methodInfo

### 現在のコード（不完全）

**[components/(kids)/KidsStampPage.tsx:346-354](components/(kids)/KidsStampPage.tsx#L346-L354)**

```typescript
// スタンプ取得方法の絵文字・ラベル・背景色（子供向け）
const methodInfo: Record<string, { emoji: string; label: string; bg: string }> = {
  qr_scan:       { emoji: "🏥", label: "びょういん に きたよ！",   bg: "bg-kids-blue/15" },
  slot_game:     { emoji: "🎰", label: "スロット で ゲット！",      bg: "bg-kids-pink/15" },
  survey_reward: { emoji: "📝", label: "アンケート ほうしゅう",    bg: "bg-kids-green/15" },
  manual_admin:  { emoji: "👨‍⚕️", label: "せんせい から",         bg: "bg-kids-yellow/30" },
};
const method = methodInfo[record.stamp_method] ?? {
  emoji: "⭐", label: "スタンプ をもらったよ", bg: "bg-gray-100",
};
```

### 問題点

1. **`"qr"` が定義されていない**
   - 現在のQR来院スタンプは `stamp_method = "qr"` を使用
   - しかし、`methodInfo` には `"qr_scan"` しかない
   - **結果：`"qr"` は fallback（⭐ スタンプ をもらったよ）になる**

2. **`"purchase_incentive"` が定義されていない**
   - 新しい購買インセンティブ用の `stamp_method`
   - **追加が必要**

3. **`"import"` が定義されていない**
   - データ移行時の `stamp_method`
   - 型定義には存在するが、`methodInfo` にはない

## 🔍 なぜ見落としたか？

### Doc 45での分析不足

1. **検索が不十分だった**

   - キッズページの実装を詳細に確認していなかった

2. **型定義に頼りすぎた**
   - 型定義で `stamp_method` の使用箇所は確認
   - しかし、表示ロジック内の**ハードコードされたマッピングオブジェクト**を見落とした

3. **大人ページとキッズページの実装差異**
   - 大人ページ：`lib/stamps.ts` の関数を使用（Doc 45で確認済み）
   - キッズページ：**独自の `methodInfo` オブジェクトを使用**（見落とし）
