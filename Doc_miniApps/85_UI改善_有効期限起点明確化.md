# UI改善実装完了 - 有効期限の起点を明確化

**実装日**: 2026-03-28
**目的**: 患者さんに有効期限の起点（マイルストーン到達時点）を明確に伝える

---

## 📋 背景

### 問題点

ユーザーテストの結果、以下の問題が発見されました：

1. **患者さんが気づかない**
   - 10個目のスタンプを取得しても、特典が付与されたことに気づかない
   - 後日アプリを開いたら既に期限切れ（歯ブラシ）

2. **「交換する」ボタンの誤解**
   - 「ボタンを押したときに有効期限が始まる」と思われている
   - 実際は既にマイルストーン到達時点で付与済み

3. **有効期限の起点が不明確**
   - 「当日限り」と表示されているが、「いつから」が不明

### 実装前の仕様確認

**重要な発見：有効期限の起点は既に正しく実装されていました**

```typescript
// lib/milestones.ts:157-173
// スタンプ付与時（マイルストーン到達時）に実行
const validUntil = calculateValidUntil(reward.validity_months);

const { data: exchange } = await supabase
  .from('reward_exchanges')
  .insert({
    user_id: userId,
    valid_until: validUntil,  // ← マイルストーン到達時に計算
    exchanged_at: new Date().toISOString(),  // ← 到達時刻を記録
    status: 'pending',
  })
```

**つまり：**
- ✅ 10個目のスタンプ取得 → その時点で有効期限計算
- ✅ 歯ブラシは取得当日の23:59:59まで有効
- ❌ UI/UXが不明確で患者さんが理解できていない

#### 1.2 ヘルパー関数の追加

```typescript
// 特典名を取得
const getRewardName = (rewardType: string): string => {
  switch (rewardType) {
    case 'toothbrush':
      return '歯ブラシ 1本';
    case 'poic':
      return 'POIC殺菌剤';
    case 'premium_menu':
      return '選べるメニュー';
    default:
      return '特典';
  }
};

// 有効期限の説明を取得
const getValidityDescription = (rewardType: string): string => {
  switch (rewardType) {
    case 'toothbrush':
      return '今日中（23:59まで）';
    case 'poic':
      return '5ヶ月間';
    case 'premium_menu':
      return '5ヶ月間';
    default:
      return '';
  }
};
```
