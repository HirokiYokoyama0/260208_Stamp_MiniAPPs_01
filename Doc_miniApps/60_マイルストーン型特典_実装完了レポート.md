#### 2. マイルストーン判定ロジックの実装

**ファイル:** [lib/milestones.ts](../lib/milestones.ts)（新規作成）

**主要な関数:**

##### `checkMilestones(oldStampCount, newStampCount): MilestoneResult[]`

スタンプ数変更時にマイルストーンを判定し、優先度ルールを適用します。

```typescript
// 例: 8スタンプ → 50スタンプに増加
checkMilestones(8, 50);
// 返り値: [
//   { milestone: 10, rewardType: 'toothbrush' },
//   { milestone: 20, rewardType: 'toothbrush' },
//   { milestone: 30, rewardType: 'toothbrush' },
//   { milestone: 40, rewardType: 'toothbrush' },
//   { milestone: 50, rewardType: 'poic' }  // ← 優先度ルールにより歯ブラシはスキップ
// ]
```

**優先度ルール:**
- 優先度: 選べる自費メニュー (3) > POIC (2) > 歯ブラシ (1)
- 50スタンプ到達時: POICのみ付与、歯ブラシはスキップ
- 100スタンプ到達時: POICのみ付与、歯ブラシはスキップ
- 300スタンプ到達時: 自費メニューのみ付与、POIC・歯ブラシはスキップ

##### `grantMilestoneReward(userId, milestone, rewardType)`

特典を自動的に `reward_exchanges` テーブルに作成します。

```typescript
// 処理内容:
// 1. milestone_rewards テーブルから特典を取得
// 2. POIC の初回/2回目を判定
// 3. 有効期限を計算
//    - 歯ブラシ: 当日限り（23:59:59まで）
//    - POIC: 5ヶ月後
//    - 自費メニュー: 5ヶ月後
// 4. reward_exchanges に INSERT（is_milestone_based = true）
// 5. milestone_history に記録
```

**有効期限の計算:**
- `validity_months = 0` → 当日の23:59:59まで
- `validity_months = 5` → 5ヶ月後
- `validity_months = NULL` → 無期限

```typescript
// ✅ マイルストーン判定を追加
import { checkMilestones, grantMilestoneReward } from "@/lib/milestones";

export async function POST(req: Request) {
  // ... スタンプ付与処理 ...

  // マイルストーン判定と特典自動付与
  const milestones = checkMilestones(currentStampCount, newStampNumber);
  const grantedRewards = [];

  for (const { milestone, rewardType } of milestones) {
    const exchange = await grantMilestoneReward(userId, milestone, rewardType);
    grantedRewards.push({ milestone, rewardType, exchangeId: exchange.id });
    console.log(`🎁 マイルストーン特典付与: ${milestone}スタンプ, ${rewardType}`);
  }

  return NextResponse.json({
    success: true,
    stampCount: newStampNumber,
    milestones: grantedRewards, // ← 付与された特典情報を返す
  });
}
```

## 🎯 動作フロー

### 例: ユーザーが8スタンプ → 50スタンプに増加

```
1. ユーザーがQRコードをスキャン（10個付与）
   ↓
2. POST /api/stamps/auto
   - oldStampCount: 8
   - newStampNumber: 18
   ↓
3. checkMilestones(8, 18)
   - 到達: 10スタンプ（歯ブラシ）
   ↓
4. grantMilestoneReward(userId, 10, 'toothbrush')
   - milestone_rewards からtoothbrushを取得
   - valid_until = 当日23:59:59
   - reward_exchanges に INSERT (is_milestone_based = true)
   - milestone_history に記録
   ↓
5. レスポンス
   {
     success: true,
     stampCount: 18,
     milestones: [
       { milestone: 10, rewardType: 'toothbrush', exchangeId: '...' }
     ]
   }
```

### 例: 48スタンプ → 50スタンプ（優先度ルール適用）

```
1. ユーザーが2個のスタンプを獲得
   ↓
2. checkMilestones(48, 50)
   - 50は10の倍数でもあり、50の倍数でもある
   - 優先度ルール適用: POIC (優先度2) > 歯ブラシ (優先度1)
   - 返り値: [{ milestone: 50, rewardType: 'poic' }]
   ↓
3. grantMilestoneReward(userId, 50, 'poic')
   - POICの初回判定: 過去にPOICを受け取っていないか確認
   - is_first_time = true なら first_time_description を使用
   - valid_until = 5ヶ月後
   - reward_exchanges に INSERT
   ↓
4. 歯ブラシは付与されない（優先度が低いため）
```
