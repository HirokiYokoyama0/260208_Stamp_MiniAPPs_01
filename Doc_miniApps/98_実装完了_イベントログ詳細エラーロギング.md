# 実装完了: イベントログ詳細エラーロギング

**実装日**: 2026-03-28
**目的**: QRスキャンイベントログが記録されない問題を調査するため、詳細なエラーログを追加

## 📋 背景

### 問題の状況
- `app_open` と `stamp_page_view` のイベントログは正常に記録される ✅
- QRスキャン関連のログ (`stamp_scan_start`, `stamp_scan_api_request`, `stamp_scan_success`) が記録されない ❌

### 調査の課題
- スマホLIFFアプリではブラウザコンソールが確認できない
- クライアントサイドのエラーが見えない
- Vercelログでサーバーサイドのエラーを確認する必要がある

## 🛠️ 実装内容

### 1. lib/analytics.ts - エラーロギング強化

#### Before（問題点）
```typescript
if (error) {
  console.error('❌ Analytics error:', error);  // エラーの詳細が不足
  return { success: false, error: error.message };
}
```

#### After（改善点）
```typescript
if (error) {
  // エラーの詳細情報を記録（サーバーサイドではVercelログに記録される）
  console.error('❌ [Analytics] Event log insertion failed:', {
    eventName,
    userId: targetUserId?.substring(0, 8) + '...',
    error: error.message,
    code: error.code,           // ← Supabaseエラーコード追加
    details: error.details,     // ← エラー詳細追加
    hint: error.hint,           // ← 修正ヒント追加
  });
  return { success: false, error: error.message };
}

// 成功時も明示的にログ出力（デバッグ用）
console.log('✅ [Analytics] Event log inserted:', {
  eventName,
  userId: targetUserId?.substring(0, 8) + '...',
  hasMetadata: !!enrichedMetadata,
  metadataKeys: enrichedMetadata ? Object.keys(enrichedMetadata) : [],
});
```

**効果**:
- Supabaseエラーの詳細（code, details, hint）が記録される
- 成功時も記録されるため、関数が実際に呼ばれているか確認可能

### 2. app/api/stamps/scan/route.ts - サーバーサイドのログ記録強化

#### Before（問題点）
```typescript
// イベントログ記録（詳細情報追加）
await logStampScanSuccess({
  stampsAdded: stamps,
  type: type,
  userId: userId,
  // ...
});
// ← エラーハンドリングなし
```

#### After（改善点）
```typescript
// イベントログ記録（詳細情報追加）
try {
  console.log('🔍 [API] スタンプスキャン成功ログを送信:', {
    stampsAdded: stamps,
    type: type,
    userId: userId.substring(0, 8) + '...',
    currentStampCount,
    newStampCount: finalStampCount,
  });

  const logResult = await logStampScanSuccess({
    stampsAdded: stamps,
    type: type,
    userId: userId,
    currentStampCount: currentStampCount,
    newStampCount: finalStampCount,
    stampHistoryId: stampData?.id,
    milestonesGranted: grantedRewards,
  });

  if (!logResult.success) {
    console.error('⚠️ [API] イベントログ記録失敗（スタンプ付与は成功）:', logResult.error);
  }
} catch (logError) {
  // イベントログ失敗してもスタンプ付与は成功しているので処理続行
  console.error('❌ [API] イベントログ記録エラー（スタンプ付与は成功）:', logError);
}
```

**効果**:
- API呼び出し前にログ出力（関数が実際に呼ばれているか確認）
- `logResult.success` をチェック（失敗時にエラーメッセージ記録）
- try-catchで例外をキャッチ（予期しないエラーも記録）
- **Vercelログで確認可能**

### 3. app/scan/page.tsx - クライアントサイドのログ記録強化

#### logStampScanStart の改善

```typescript
// Before
await logStampScanStart({
  userId: profile?.userId,
  deviceType,
});

// After
try {
  console.log('📱 [Scan] QRスキャン開始ログを送信:', {
    userId: profile?.userId?.substring(0, 8) + '...',
    deviceType,
  });

  const scanStartResult = await logStampScanStart({
    userId: profile?.userId,
    deviceType,
  });

  if (!scanStartResult.success) {
    console.error('⚠️ [Scan] QRスキャン開始ログ送信失敗:', scanStartResult.error);
  }
} catch (err) {
  console.error('❌ [Scan] QRスキャン開始ログ送信エラー:', err);
}
```

#### logStampScanApiRequest の改善

```typescript
// Before
await logStampScanApiRequest({
  userId: profile?.userId,
  qrRawValue: qrValue,
  qrParsed: payload,
  requestPayload,
});

// After
try {
  console.log('📱 [Scan] APIリクエストログを送信:', {
    userId: profile?.userId?.substring(0, 8) + '...',
    qrType: payload.type,
    qrStamps: payload.stamps,
  });

  const apiRequestResult = await logStampScanApiRequest({
    userId: profile?.userId,
    qrRawValue: qrValue,
    qrParsed: payload,
    requestPayload,
  });

  if (!apiRequestResult.success) {
    console.error('⚠️ [Scan] APIリクエストログ送信失敗:', apiRequestResult.error);
  }
} catch (err) {
  console.error('❌ [Scan] APIリクエストログ送信エラー:', err);
}
```

**効果**:
- 呼び出し前にログ出力（関数が実際に呼ばれているか確認）
- 成功/失敗の結果をログ記録
- ブラウザコンソールで確認可能（開発環境）

## 🔍 デバッグ方法

### 1. Vercelログでサーバーサイドを確認

1. **Vercelダッシュボード** にアクセス
   - https://vercel.com/
   - プロジェクトを選択
   - 「Logs」タブ

2. **確認すべきログ**

#### 成功時のログ
```
✅ QRスキャンスタンプ登録成功: User xxx, Type: premium, Stamps: +15個
🔍 [API] スタンプスキャン成功ログを送信: { stampsAdded: 15, type: 'premium', ... }
✅ [Analytics] Event log inserted: { eventName: 'stamp_scan_success', ... }
```

#### 失敗時のログ
```
❌ [Analytics] Event log insertion failed: {
  eventName: 'stamp_scan_success',
  error: '...',
  code: '42501',  // RLS violation などのエラーコード
  details: '...',
  hint: '...'
}
```

### 2. ブラウザコンソールでクライアントサイドを確認（開発環境）

```
📱 [Scan] QRスキャン開始ログを送信: { userId: 'U123456...', deviceType: 'iPhone' }
✅ [Analytics] Event log inserted: { eventName: 'stamp_scan_start', ... }
```

### 3. Supabase SQL Editorで確認

```sql
-- 最新のイベントログ
SELECT
  event_name,
  user_id,
  created_at,
  metadata->>'device_type' as device_type,
  metadata->>'stamps_added' as stamps_added
FROM event_logs
WHERE event_name LIKE 'stamp_scan%'
ORDER BY created_at DESC
LIMIT 10;
```
