# スタッフ操作：削除イベントログ仕様

**作成日**: 2026-03-28
**目的**: スタッフによる「本日のQRスキャンを削除」操作を記録し、不正操作の監視と問題調査を可能にする

## 📋 背景・目的

### なぜ必要か

1. **不正操作の検知**
   - スタッフが頻繁に削除操作を行っていないか監視
   - 特定のユーザーに対して不当に削除していないか確認

2. **問題調査**
   - 「スタンプが消えた」とユーザーから報告があった場合
   - スタッフ操作で削除されたのか、システムの問題か判別

3. **運用状況の把握**
   - テスト運用でどれくらい削除操作が使われているか
   - 本番運用前に操作頻度を確認

## 🎯 記録すべき情報

### イベント名

- `stamp_delete_today_qr` - 本日のQRスキャンを削除

### メタデータ

```json
{
  "target_user_id": "U5c70cd61...",          // 対象ユーザーのLINE ID
  "target_user_name": "山田太郎",            // 対象ユーザーの表示名
  "target_ticket_number": "123456",         // 対象ユーザーの会員証番号

      "stamp_number": 65,

```

## 📝 実装内容

### 1. lib/analytics.ts に新規関数を追加

```typescript
/**
 * 本日のQRスキャン削除ログ（スタッフ操作）
 */
export const logStampDeleteTodayQR = (params: {
  userId?: string;  // 操作したスタッフ（通常は対象ユーザーと同じ）
  targetUserId: string;
  targetUserName?: string;
  targetTicketNumber?: string;
  deletedCount: number;
  deletedStamps: number;
  deletedRecords?: Array<{
    id: string;
    visit_date: string;
    stamp_number: number;
    amount: number;
    stamp_method: string;
  }>;
}) => {
  return logEvent({
    eventName: 'stamp_delete_today_qr',
    userId: params.userId || params.targetUserId,
    metadata: {
      target_user_id: params.targetUserId,
      target_user_name: params.targetUserName,
      target_ticket_number: params.targetTicketNumber,
      deleted_count: params.deletedCount,
      deleted_stamps: params.deletedStamps,
      deleted_records: params.deletedRecords,
      staff_initiated: true,
    },
  });
};
```

## 🔍 調査用SQL

### 1. 削除操作の履歴を確認

```sql
-- 最近の削除操作を確認
SELECT
  el.created_at,
  el.metadata->>'target_user_name' as user_name,
  el.metadata->>'target_ticket_number' as ticket,
  (el.metadata->>'deleted_count')::int as deleted_count,
  (el.metadata->>'deleted_stamps')::int as deleted_stamps,
  el.metadata->'deleted_records' as records
FROM event_logs el
WHERE el.event_name = 'stamp_delete_today_qr'
  AND el.created_at >= current_date - interval '30 days'
ORDER BY el.created_at DESC;
```

### 2. 特定ユーザーの削除履歴

```sql
-- 特定ユーザーに対する削除操作を確認
SELECT
  el.created_at,
  (el.metadata->>'deleted_count')::int as deleted_count,
  (el.metadata->>'deleted_stamps')::int as deleted_stamps
FROM event_logs el
WHERE el.event_name = 'stamp_delete_today_qr'
  AND el.metadata->>'target_user_id' = 'U5c70cd61...'  -- ユーザーID
ORDER BY el.created_at DESC;
```

### 3. 削除操作の頻度分析

```sql
-- 日別の削除操作回数
SELECT
  DATE(created_at) as date,
  COUNT(*) as delete_count,
  SUM((metadata->>'deleted_stamps')::int) as total_stamps_deleted
FROM event_logs
WHERE event_name = 'stamp_delete_today_qr'
  AND created_at >= current_date - interval '30 days'
GROUP BY DATE(created_at)
ORDER BY date DESC;
```

### 4. スタンプ消失問題の調査

```sql
-- ユーザーからの報告：「スタンプが消えた」
-- → スタッフ操作で削除されたか確認

-- Step 1: ユーザーIDまたは会員証番号を取得
SELECT id, display_name, ticket_number
FROM profiles
WHERE display_name LIKE '%山田%'
   OR ticket_number = '123456';

-- Step 2: そのユーザーのスタンプスキャン履歴を確認
SELECT
  created_at,
  event_name,
  CASE
    WHEN event_name = 'stamp_scan_success' THEN '✅ スキャン成功'
    WHEN event_name = 'stamp_delete_today_qr' THEN '🗑️ スタッフが削除'
  END as action,
  metadata
FROM event_logs
WHERE (user_id = 'U5c70cd61...' OR metadata->>'target_user_id' = 'U5c70cd61...')
  AND event_name IN ('stamp_scan_success', 'stamp_delete_today_qr')
  AND created_at >= current_date - interval '7 days'
ORDER BY created_at DESC;
```

## 📊 運用での活用例

### ケース1: ユーザーから「スタンプが消えた」と報告

```
Step 1: ユーザーのイベントログを確認
  ↓
【パターンA】stamp_delete_today_qr が記録されている
  → スタッフが削除した（意図的 or 誤操作）
  → スタッフに確認・指導

【パターンB】削除ログがない
  → システムの問題（トリガーのバグなど）
  → 開発者が調査
```

### ケース2: テスト運用での削除頻度確認

```sql
-- 過去1ヶ月の削除操作回数
SELECT COUNT(*) as total_deletes
FROM event_logs
WHERE event_name = 'stamp_delete_today_qr'
  AND created_at >= current_date - interval '30 days';

-- 頻度が高い場合
→ 本番運用前にUI/UXの改善が必要かも
→ 削除操作をより慎重にする（確認ダイアログ追加など）
```

### ケース3: 不正操作の検知

```sql
-- 特定のスタッフ（または不審なユーザー）が頻繁に削除している
SELECT
  metadata->>'target_user_name' as deleted_user,
  COUNT(*) as delete_count
FROM event_logs
WHERE event_name = 'stamp_delete_today_qr'
  AND created_at >= current_date - interval '7 days'
GROUP BY metadata->>'target_user_name'
HAVING COUNT(*) > 5  -- 1週間で5回以上削除
ORDER BY delete_count DESC;
```
