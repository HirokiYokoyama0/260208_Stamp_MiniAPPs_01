## 🎯 QRコードURL仕様（確定版）

### カメラ用QR（LIFF URL）

**重要**: `/auto-stamp` は省略可能。ルートページが自動リダイレクトします。

#### 優良患者様用（15個）
```
https://liff.line.me/{LIFF_ID}?action=stamp&type=qr&amount=15&location=entrance
```

#### 通常患者様用（10個）
```
https://liff.line.me/{LIFF_ID}?action=stamp&type=qr&amount=10&location=entrance
```

#### 購買インセンティブ用（5個）
```
https://liff.line.me/{LIFF_ID}?action=stamp&type=purchase&amount=5&location=shop
```

### アプリ内スキャン用QR（JSON形式）

#### 優良患者様用（15個）
```json
{"type":"premium","stamps":15}
```

#### 通常患者様用（10個）
```json
{"type":"regular","stamps":10}
```

#### 購買インセンティブ用（5個）
```json
{"type":"purchase","stamps":5}
```

## 🔧 技術的な重要ポイント

### 1. ルートページリダイレクト機能

**実装**: `app/page.tsx`

```typescript
if (action === 'stamp' && (type === 'qr' || type === 'purchase') && amount) {
  // /auto-stamp にリダイレクト
}
```

**効果**: QRコードのURLに `/auto-stamp` が無くても動作する

### 2. 購買インセンティブの判定

**location パラメータで判定:**
- `location=entrance` → 来院QR（1日1回制限）
- `location=shop` → 購買インセンティブ（無制限）

**type パラメータで判定:**
- `type=qr` → 来院QR
- `type=purchase` → 購買インセンティブ

### 3. stamp_method の値

| QR種別 | stamp_method | 1日制限 |
|--------|-------------|---------|
| 優良患者様 | `"qr"` | あり |
| 通常患者様 | `"qr"` | あり |
| 購買インセンティブ | `"purchase_incentive"` | **なし** |
