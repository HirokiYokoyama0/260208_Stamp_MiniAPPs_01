#### 1.2 有効期限チェック関数

```typescript
// 有効期限チェック関数
const checkExpiration = (exchange: RewardExchange | null): { isExpired: boolean; daysRemaining: number | null } => {
  if (!exchange || !exchange.valid_until) {
    return { isExpired: false, daysRemaining: null };
  }

  const now = new Date();
  const validUntil = new Date(exchange.valid_until);
  const isExpired = validUntil < now;

  // 残り日数を計算
  const diffMs = validUntil.getTime() - now.getTime();
  const diffDays = Math.ceil(diffMs / (1000 * 60 * 60 * 24));

  return {
    isExpired,
    daysRemaining: isExpired ? null : diffDays,
  };
};

// 日付フォーマット関数
const formatValidUntil = (isoString: string): string => {
  const date = new Date(isoString);
  return date.toLocaleDateString('ja-JP', {
    year: 'numeric',
    month: '2-digit',
    day: '2-digit',
  });
};
```

### 2. バックエンド側（Phase 1 必須）

#### ファイル: `app/api/rewards/exchange/route.ts`

#### 2.1 交換申請時の期限チェック

既存の `pending` チェックに有効期限チェックを追加しました。

```typescript
// 4. pending チェック（重複防止） + 有効期限チェック
console.log('🔍 既存のpending申請をチェック中...');
const { data: existingPending } = await supabase
  .from("reward_exchanges")
  .select("id, milestone_reached, valid_until, status")
  .eq("user_id", userId)
  .eq("reward_id", rewardId)
  .eq("milestone_reached", milestone || fullStamps)
  .eq("status", "pending")
  .maybeSingle();

if (existingPending) {
  // 既存のpending申請がある場合、有効期限をチェック
  if (existingPending.valid_until) {
    const now = new Date();
    const validUntil = new Date(existingPending.valid_until);

    if (validUntil < now) {
      // 期限切れの場合、自動的にexpiredに更新
      console.log('⏰ 既存の申請が期限切れのため、expired に更新します');
      await supabase
        .from("reward_exchanges")
        .update({ status: "expired" })
        .eq("id", existingPending.id);

      // 期限切れ処理後、新規申請として続行
    } else {
      // まだ有効な場合は重複エラー
      return NextResponse.json(
        {
          success: false,
          message: "この特典は申請中です。受付でお受け取りください。",
          error: "Already pending",
        },
        { status: 400 }
      );
    }
  }
}
```

**動作:**
- 同じマイルストーンで既に `pending` の申請がある場合、有効期限をチェック
- 期限切れの場合は自動的に `status = 'expired'` に更新し、新規申請を許可
- まだ有効な場合は重複エラーを返す

## 📝 注意事項

### 1. タイムゾーン

- `valid_until` は **UTC** で保存されている
- フロントエンドでの日付比較は `new Date()` で自動的にブラウザのタイムゾーンに変換される
- 日本時間（JST）での表示は `toLocaleDateString('ja-JP')` で対応

### 2. 歯ブラシの「当日限り」

- `valid_until` は「翌日0時」に設定される（例: `2026-03-28T23:59:59+09:00`）
- 3月29日になると自動的に期限切れとなる

### 3. 期限切れでも削除はしない

- `status = 'expired'` として残す
- 理由:
  - 特典付与履歴として残す
  - スタッフが状況を確認できる
  - 統計データに使用

### 4. データベースのRLS設定

期限切れの自動更新が正しく動作するには、Supabaseの Row Level Security (RLS) で以下のポリシーが設定されている必要があります:

```sql
-- reward_exchanges テーブルのUPDATEポリシー
-- ユーザーは自分の交換履歴を更新できる
CREATE POLICY "Users can update their own exchanges"
ON reward_exchanges
FOR UPDATE
USING (auth.uid() = user_id);
```
