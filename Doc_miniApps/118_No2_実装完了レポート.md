## 📋 実装内容

### 問題の概要

**iOS 18.6.2以降で登録QR（qanvaqr.com）をスキャンすると、Safari（外部ブラウザ）で開かれてしまい、LIFFアプリが正常に動作しない**

- 該当ユーザー: 3人（iOS 18ユーザー）
- 症状: LIFF初期化を53秒繰り返す → タイムアウト
- 原因: qanvaqr.comのQRコードがLIFF URLではないため

### 実装した対策

**外部ブラウザ判定と誘導UIの実装**

1. LIFF初期化後に`liff.isInClient()`で判定
2. 外部ブラウザの場合、誘導画面を表示
3. LINEアプリで開き直すよう案内

### 2. components/layout/AppLayout.tsx（修正）

**追加した処理**:

#### インポート追加
```typescript
import liff from "@line/liff";
import { ExternalBrowserWarning } from "@/components/ExternalBrowserWarning";
```

#### ステート追加
```typescript
const [isExternalBrowser, setIsExternalBrowser] = useState(false);
const [isLiffChecked, setIsLiffChecked] = useState(false);
```

#### 判定ロジック追加
```typescript
useEffect(() => {
  if (!isLoading && !isLiffChecked) {
    try {
      if (!liff.isInClient()) {
        console.warn("⚠️ [AppLayout] 外部ブラウザで開かれています");
        setIsExternalBrowser(true);
      } else {
        console.log("✅ [AppLayout] LINEアプリ内で開かれています");
        setIsExternalBrowser(false);
      }
    } catch (error) {
      console.error("❌ [AppLayout] LIFF判定エラー:", error);
      setIsExternalBrowser(false); // エラー時は通常通り表示
    }
    setIsLiffChecked(true);
  }
}, [isLoading, isLiffChecked]);
```

#### 誘導画面表示ロジック
```typescript
if (isLiffChecked && isExternalBrowser) {
  return <ExternalBrowserWarning />;
}

// 通常はchildrenを表示
return (
  <ViewModeProvider>
    {/* 既存のレイアウト */}
  </ViewModeProvider>
);
```
