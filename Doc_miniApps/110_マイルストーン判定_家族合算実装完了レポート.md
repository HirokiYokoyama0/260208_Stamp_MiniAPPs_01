# マイルストーン判定 家族合算実装完了レポート

**実装完了日:** 2026-04-04
**ステータス:** ✅ 実装完了・Vercelデプロイ済み
**関連Issue:** 特典タブでの重複マイルストーン問題（10スタンプ特典が2つ表示）

---

## 📋 実装背景

### 発見された問題

1. **重複マイルストーン問題**
   - 症状: 10スタンプマイルストーンが2つ表示される
   - 原因: `checkMilestones()` が既存の付与済みマイルストーンをチェックしていなかった
   - タイムライン:
     ```
     1. スタッフ操作: 999個 → 0個（履歴全削除）
     2. 何らかの操作で10スタンプ到達 → 10スタンプ特典付与（1回目）
     3. 管理ダッシュボードで status を completed に変更
     4. カメラQR: 0個 → 15個
        → checkMilestones(0, 15) が 10 を返す
        → 既存特典をチェックしないため、10スタンプ特典を再付与（2回目）
     ```

2. **家族スタンプ合算の未実装**
   - 仕様書: マイルストーン判定は家族スタンプ合算で行うべき
   - 実装: 個人スタンプ数のみで判定していた
   - 影響: 家族でスタンプを集めてもマイルストーン特典が付与されない

## ✅ 実装内容

### 1. 重複マイルストーン防止機能

#### 修正ファイル: `lib/milestones.ts`

**変更内容:**
- `checkMilestones()` を async 関数に変更
- `userId` パラメータを追加
- Supabase から既存マイルストーン特典を取得
- 既に付与済みのマイルストーンを除外して返す

**修正前:**
```typescript
export function checkMilestones(
  oldStampCount: number,
  newStampCount: number
): MilestoneResult[] {
  // マイルストーン計算のみ、重複チェックなし
  const milestones = calculateMilestones(oldStampCount, newStampCount);
  return milestones.map(m => ({
    milestone: m,
    rewardType: getRewardTypeForMilestone(m)
  }));
}
```

**修正後:**
```typescript
export async function checkMilestones(
  userId: string,  // ← 新規追加
  oldStampCount: number,
  newStampCount: number
): Promise<MilestoneResult[]> {
  // マイルストーン計算
  const allMilestones = calculateMilestones(oldStampCount, newStampCount);
  const uniqueMilestones = [...new Set(allMilestones)].sort((a, b) => a - b);

  if (uniqueMilestones.length === 0) {
    return [];
  }

  // 🆕 既存マイルストーン特典をチェック
  const { data: existingRewards } = await supabase
    .from('reward_exchanges')
    .select('milestone_reached')
    .eq('user_id', userId)
    .eq('is_milestone_based', true)
    .in('milestone_reached', uniqueMilestones);

  const existingMilestones = new Set(
    existingRewards?.map((r) => r.milestone_reached) || []
  );

  // 🆕 既存マイルストーンを除外
  const results: MilestoneResult[] = [];
  for (const milestone of uniqueMilestones) {
    if (!existingMilestones.has(milestone)) {
      const rewardType = getRewardTypeForMilestone(milestone);
      results.push({ milestone, rewardType });
      console.log(`✅ [Milestones] 新規マイルストーン: ${milestone}スタンプ (${rewardType})`);
    } else {
      console.log(`⏭️ [Milestones] スキップ: ${milestone}スタンプ (既に付与済み)`);
    }
  }

  return results;
}
```

**効果:**
- スタッフ操作後の復活時に重複付与されない
- 10スタンプ特典が2つ発行される問題が解決
- 全マイルストーンで重複チェックが有効

### 2. 家族スタンプ合算機能

#### 仕様決定

**マイルストーン判定の基準:**
- `family_id = NULL` → 個人スタンプ数で判定
- `member_count = 1` → 個人スタンプ数で判定（家族合算=個人）
- `member_count >= 2` → 家族合算スタンプ数で判定

**遡及適用:**
- **決定:** A. 一気に付与する（意図通り）
- **理由:** 既存の家族ユーザーが不公平にならないよう、次回スキャン時に未付与マイルストーンを一括付与
- **影響:** 既存の複数人家族ユーザーは次回スキャン時に複数の特典を一度に受け取る可能性がある

**スマホなし子供への特典付与:**
- **決定:** A. 親のアカウントに付与
- **理由:** `line_user_id = NULL` の子供アカウントはLINEミニアプリにアクセスできない
- **実装:** スキャンしたユーザー（親）に特典を付与（現在の設計で自動的に実現）

#### 実装ファイル

**修正した3つのAPI:**
1. `app/api/stamps/route.ts` - ホームボタンスキャン
2. `app/api/stamps/scan/route.ts` - アプリ内スキャン
3. `app/api/stamps/auto/route.ts` - カメラ用QR

**実装内容（共通）:**

## 🔍 重複チェックの詳細動作

### シナリオ: スタッフ操作後の復活

```
タイムライン:
1. 初期状態: 999個
2. スタッフ操作: 999個 → 0個（stamp_history 全削除）
3. QRスキャン: 0個 → 10個
   - checkMilestones(userId, 0, 10)
   - reward_exchanges に milestone_reached = 10 を INSERT
4. 管理ダッシュボード: status を 'completed' に変更
5. QRスキャン: 10個 → 25個
   - checkMilestones(userId, 10, 25)
   - 計算結果: [20]
   - 既存チェック: milestone_reached = 10 は既に存在 → スキップ
   - milestone_reached = 20 は未存在 → 付与
   - 返り値: [{ milestone: 20, rewardType: 'toothbrush' }]
```

**修正前（バグ）:**
- 0個 → 25個 でスキャンした場合、10スタンプと20スタンプの両方が付与される
- 10スタンプが重複して2つ表示される

**修正後（正常）:**
- 既存の10スタンプ特典をチェック
- 20スタンプのみが付与される
- 重複なし

## 📝 データベース仕様

### 使用するビュー: `family_stamp_totals`

**定義:**
```sql
CREATE OR REPLACE VIEW family_stamp_totals AS
SELECT
  family_id,
  COUNT(*) AS member_count,
  SUM(stamp_count) AS total_stamp_count
FROM profiles
WHERE family_id IS NOT NULL
GROUP BY family_id;
```

**カラム:**
- `family_id`: 家族ID
- `member_count`: 家族メンバー数
- `total_stamp_count`: 家族の合計スタンプ数

### 使用するテーブル: `reward_exchanges`

**関連カラム:**
- `milestone_reached`: マイルストーン到達数（10, 20, 30, ...）
- `is_milestone_based`: マイルストーン型特典フラグ（true/false）
- `user_id`: ユーザーID（重複チェック用）

## 🎯 重要な仕様決定

### 1. 遡及適用ポリシー

**決定:** 既存家族ユーザーに一括付与する

**理由:**
- 仕様書通りの実装を徹底
- 既存ユーザーへの公平性を保つ
- 実装がシンプル（特別な移行処理不要）

**注意点:**
- ユーザーは次回スキャン時に複数の特典を一度に受け取る可能性がある
- LINEミニアプリのUI上、複数の特典が一度に表示される
- 管理ダッシュボード側で事前に状況を把握しておくことを推奨

### 2. スマホなし子供の特典付与先

**決定:** 親のアカウントに付与する

**理由:**
- `line_user_id = NULL` の子供はLINEミニアプリにアクセスできない
- 家族の代表（親）が特典を受け取り、実際の利用時に家族全員で使える
- 現在の実装で自動的に実現される（スキャンしたユーザーに付与）

**実装:**
- 特別な処理は不要
- `grantMilestoneReward(userId, ...)` の userId がスキャンしたユーザー

### 3. 家族判定ロジック

**ロジック:**
```typescript
if (profileData?.family_id && familyTotal?.member_count >= 2) {
  // 家族合算を使用
  effectiveStampCountOld = oldFamilyTotal;
  effectiveStampCountNew = familyTotal.total_stamp_count;
} else {
  // 個人スタンプを使用
  effectiveStampCountOld = currentStampCount;
  effectiveStampCountNew = nextStampNumber;
}
```

**理由:**
- `family_id = NULL` → 単身ユーザー、個人判定
- `member_count = 1` → 1人家族、合算=個人なので個人判定でOK
- `member_count >= 2` → 複数人家族、家族合算判定
